# 农业合作社-月底集中处理种植任务系统

围绕种植任务被退回的卡点（材料缺失、时限逾期、状态不一致）设计，支持三类身份协作处理种植任务全流程。

## 技术栈

- **前端**: SolidStart (端口 3001)
- **后端**: NestJS (端口 8001)
- **数据库**: SQLite (本地文件 `backend/data/agri_coop.db`)

## 快速启动

### 1. 安装依赖

```bash
# 后端
cd backend
npm install

# 前端
cd ../frontend
npm install
```

### 2. 初始化 SQLite 并导入演示数据

```bash
cd backend
npx ts-node src/seed.ts
```

这会自动创建 `data/agri_coop.db` 并插入演示账号和演示单据。如数据库已有数据则跳过。

### 3. 启动后端

```bash
cd backend
npm run start:dev
```

后端监听 http://localhost:8001/api

### 4. 启动前端

```bash
cd frontend
npm run dev
```

前端访问 http://localhost:3001

## 异常入口

- **越权归档**: 用农技员账号尝试归档 ZZ-202606-0005 → 返回 `UNAUTHORIZED_ROLE: 农技员不能替合作社主任归档`
- **重复提交**: 用相同 version 提交 ZZ-202606-0003 → 返回 `VERSION_CONFLICT: 版本冲突`
- **状态冲突**: 对已归档的 ZZ-202605-0001 执行操作 → 返回 `INVALID_STATUS_TRANSITION`
- **缺证据**: 不提供 evidence 处理 ZZ-202606-0002 → 返回 `MISSING_EVIDENCE: 开始处理需要提供处理依据`
- **越权推进**: 田间管理员尝试归档 → 返回 `FORBIDDEN_ACTION: 田间管理员不能跳过处理环节`
- **缺材料归档**: 有未审批农资领用时尝试归档 → 返回 `MISSING_MATERIAL`

## 模块说明

### 种植计划（PlantingTask）
核心模块，任务状态流转: 待分派 → 已分派 → 处理中 → 已转办 → 已回访 → 已归档，任意环节可退回补正。

### 农资领用（MaterialRequisition）
关联种植任务的物资申请，待审批/已审批/已驳回三种状态。归档时检查是否存在未审批农资。

### 田间记录（FieldRecord）
关联种植任务的田间操作记录，类型包括播种、施肥、病虫害防治、收获、巡检。

### 审计日志（AuditLog）
记录操作人、操作前后状态、失败原因，支持按任务、操作人、操作类型筛选。

### 到期预警（OverdueQueue）
按截止日期分为正常/临期(3天内)/逾期，逾期批量推进逐条给拦截结果。

### 导出（Export）
导出当前筛选结果的 CSV 文件，首行标注数据时间。

## 权限校验

提交处理结果时检查以下条件:

1. **资料缺失**: 归档时检查是否有未审批的农资领用
2. **状态冲突**: 只允许按流程推进，不允许跳步
3. **越权推进**: 农技员不能归档，田间管理员不能跳处理环节
4. **超期未处理**: 逾期任务在预警队列中标记，批量推进时逐条校验
5. **版本冲突**: 提交时检查版本号，防止并发修改

前后端错误码一致，不返回通用失败信息。
