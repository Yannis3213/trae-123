# 二手车交易平台 · 月底集中处理车源上架单系统

## 技术栈

- **前端**：Vite + React + React Router + TanStack Table，端口 `3106`
- **后端**：Python Django + Django Ninja，端口 `8106`
- **数据库**：SQLite（项目内 `backend/db.sqlite3`）

## 快速启动

```bash
# 1. 安装后端依赖
cd backend
pip install django django-ninja django-cors-headers

# 2. 初始化数据库（建表 + 演示数据）
python manage.py migrate
python init_db.py

# 3. 启动后端（端口 8106）
python manage.py runserver 0.0.0.0:8106 --noreload

# 4. 安装前端依赖（新终端）
cd frontend
npm install

# 5. 启动前端（端口 3106，API 代理到 8106）
npm run dev
```

浏览器打开 http://localhost:3106

## SQLite 初始化说明

`init_db.py` 会自动执行以下操作：

1. 清空旧数据（Operator、VehicleListingApplication、ProcessingRecord、AuditNote、Attachment）
2. 创建 4 个演示操作人
3. 创建 8 条演示车源上架单（4 类场景）
4. 创建对应的处理记录、审计备注和附件

数据库文件：`backend/db.sqlite3`

如需重置数据，重新运行 `python init_db.py` 即可。

## 异常样例

### 越权操作

评估师尝试复核（门店经理权限）：
```
POST /api/listings/{id}/review
→ 403 "车源上架单CJ...001：当前角色'评估师'无权执行'复核'操作"
```

### 状态冲突

对已办结的上架单再次提交：
```
POST /api/listings/{id}/submit
→ 400 "车源上架单CJ...001：当前状态为'办结'，无法提交"
```

### 版本冲突（乐观锁）

两个窗口同时操作同一上架单，页面版本为 2，但服务端已变为 3：
```
POST /api/listings/{id}/submit { status: "DRAFT", version: 2 }
→ 409 "车源上架单CJ...001：版本冲突：页面版本2与服务端版本3不一致，请刷新后重试"
```

### 缺证据请求

提交时无挂牌确认证据且未填写原因：
```
POST /api/listings/ { ..., has_listing_evidence: false, missing_evidence_reason: "" }
→ 400 "缺挂牌确认证据时必须填写原因"
```

批量提交缺证据的上架单：
```
→ 逐条失败原因："车源上架单CJ...003：缺挂牌确认证据，无法推进到待处理"
```

### 退回时未填原因

门店经理退回补正但未填写退回原因：
```
POST /api/listings/{id}/review { action: "return", reject_reason: "" }
→ 400 "车源上架单CJ...001：退回时必须填写退回原因"
```

### 非本人操作

车源顾问尝试提交他人创建的上架单：
```
POST /api/listings/{id}/submit
→ 403 "车源上架单CJ...001：只有申请人本人才能提交"
```

## 业务流程

### 角色与操作

| 角色 | 可执行操作 |
|------|-----------|
| 车源顾问 | 创建上架单、提交、补正 |
| 评估师 | 评估处理 |
| 门店经理 | 复核通过、退回补正 |

### 状态流转

```
草稿 ──提交──→ 待处理（有证据）/ 待补正（缺证据）
待补正 ──补正──→ 待处理
退回补正 ──补正──→ 待处理
待处理 ──评估处理──→ 复核中
复核中 ──复核通过──→ 办结
复核中 ──退回补正──→ 退回补正
```

### 页面标签映射

页面仅展示 3 种标签，不暴露内部状态：

| 内部状态 | 页面标签 |
|----------|----------|
| 草稿、待补正、退回补正、待处理 | 待补正 |
| 处理中、复核中 | 复核中 |
| 办结 | 办结 |

### 三大流程模块

- **车源录入**：品牌、型号、车架号、车牌号等基本信息
- **检测评估**：评估师填写评估结果
- **挂牌确认**：挂牌确认证据状态、缺证据原因、补正说明、复核结果、退回原因

## 到期预警

列表页「到期预警」Tab 展示三组，严格分离：

| 分组 | 条件 | 颜色 |
|------|------|------|
| 逾期 | 截止时间已过 | 红色 |
| 临期 | 截止时间在 3 天内 | 橙色 |
| 正常 | 截止时间在 3 天以上或无截止时间 | 绿色 |

节点超时算到责任人。逾期批量推进（评估师操作）逐条拦截，记录超时责任人。

## 审计日志

每条操作记录包含：

- 操作人、角色
- 前状态 → 后状态
- 备注
- 失败原因（如有）

详情页「审计轨迹」Tab 可查看完整操作记录和审计备注。

## 批量处理

1. 列表页勾选多条上架单
2. 点击「批量处理」
3. 选择操作类型（按角色限制）
4. 提交后逐条显示结果：
   - ✓ 成功 + 具体原因（如"状态变更为待处理"）
   - ✗ 失败 + 具体原因（如"缺挂牌确认证据，无法推进到待处理"）

## 导出

点击「导出」按钮，输出当前筛选结果的 CSV 文件：

- 文件首行标注数据时间
- 只包含当前筛选条件下的记录

## 安全机制

后端在每个状态变更操作前校验：

1. **角色权限**：只允许对应角色执行操作
2. **当前处理人**：提交和补正只允许申请人本人
3. **状态校验**：只允许从合法状态流转
4. **版本校验**：比较页面状态与后端记录，防止静默覆盖
5. **必填证据**：缺挂牌确认证据时必须填写原因

绕开页面直接调 API 时，后端同样拦截以上所有异常。
