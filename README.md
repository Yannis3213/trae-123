# 服装加工厂 - 月底集中处理打样任务系统

## 项目简介

基于 Next.js App Router + Rust Actix Web + SQLite 的打样任务管理系统，支持多角色权限管理、任务流转、批量处理、审计轨迹等功能。

## 技术栈

- **前端**: Next.js 14 (App Router) + TypeScript + React 18
- **后端**: Rust + Actix Web + SQLx
- **数据库**: SQLite
- **端口**: 前端 3003，后端 8003

## 角色配置

| 角色 | 说明 | 默认用户 |
|------|------|----------|
| 打样登记员 (sampling_registrar) | 发起/补正打样任务 | 张登记 |
| 打样审核主管 (sampling_supervisor) | 分派、审核、退回 | 李主管 |
| 服装加工厂复核负责人 (factory_reviewer) | 复核、归档 | 王厂长 |

## 状态流转

```
待分派 → 已分派 → 待审核 → 已审核 → 待复核 → 已归档
  ↓         ↓         ↓         ↓         ↓
  └─── 退回补正 ───┘         └─── 退回 ────┘
         ↓
       已退回 → 补正 → 待审核
```

## 快速开始

### 后端启动

```bash
cd backend

# 安装依赖并构建
cargo build

# 初始化示例数据（可选）
cargo run --bin seed_data

# 启动服务（默认端口 8003）
cargo run
```

环境变量配置（`.env` 文件）：
```
PORT=8003
DATABASE_URL=sqlite:./data/sampling_tasks.db
FRONTEND_URL=http://localhost:3003
```

### 前端启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器（默认端口 3003）
npm run dev
```

前端会通过代理将 `/api` 请求转发到 `http://localhost:8003/api`。

## API 接口

### 基础路径
`http://localhost:8003/api`

### 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/tasks` | 获取任务列表 |
| GET | `/tasks/{id}` | 获取任务详情（含处理记录、审计备注、附件、异常原因） |
| POST | `/tasks` | 创建打样任务 |
| POST | `/tasks/process` | 处理任务（分派、审核、退回、复核、归档、补正等） |
| POST | `/tasks/batch-process` | 批量处理任务 |
| GET | `/tasks/statistics` | 获取统计数据 |
| GET | `/tasks/{id}/records` | 获取处理记录 |

### 操作类型 (action)

- `assign` - 分派任务（审核主管）
- `review` - 审核通过（审核主管）
- `return` - 退回补正（审核主管/复核负责人）
- `verify` - 复核确认（复核负责人）
- `archive` - 归档（复核负责人，需大货排产证据）
- `rectify` - 补正提交（登记员）
- `reassign` - 转办任务（审核主管）
- `add_evidence` - 补充证据（登记员/审核主管）

### 校验规则

- **角色校验**: 每个操作只能由对应角色执行
- **状态校验**: 操作需符合状态流转规则
- **版本校验**: 提交版本需与当前版本一致，防止并发冲突
- **证据校验**: 归档/复核操作必须提供大货排产证据
- **越权校验**: 不同角色只能查看权限范围内的任务

## 功能特性

### 列表页
- 角色切换模拟
- 多条件筛选（状态、优先级、到期状态、关键词）
- 批量选择和批量操作
- 到期预警分组（正常、临期、逾期）
- 统计概览卡片
- 异常标签展示

### 详情页
- 任务基本信息
- 办理操作（按角色显示可用操作）
- 处理记录时间线
- 审计备注
- 附件管理
- 异常原因记录
- 版本号追踪

### 批量处理
- 逐条执行，逐条返回结果
- 成功/失败原因明确
- 失败不中断整体执行

### 审计追踪
- 完整的状态变更记录
- 操作人、操作时间、版本号
- 审计备注独立记录

## CORS 配置

- `http://localhost:3003`
- `http://127.0.0.1:3003`

可通过环境变量 `FRONTEND_URL` 配置。
