# 供应链金融平台 - 月底集中处理融资申请单系统

基于 **SvelteKit** 前端 + **Rust Rocket** 后端 + **SQLite** 本地数据库的全栈应用。

> 前端端口：`3109` | 后端端口：`8109`

## 目录结构

```
.
├── backend/                 # Rust Rocket 后端
│   ├── src/
│   │   ├── main.rs         # 主入口与路由
│   │   ├── models.rs       # 数据模型
│   │   └── db.rs           # 数据库访问层
│   ├── data/               # SQLite 数据库文件目录
│   ├── init.sql            # 数据库初始化脚本
│   ├── Cargo.toml          # Rust 依赖配置

├── frontend/               # SvelteKit 前端
│   ├── src/
│   │   ├── routes/         # 页面路由
│   │   └── lib/            # 工具库
│   ├── package.json

│   └── vite.config.js      # Vite 配置（端口 3109 + 代理到 8109）
└── README.md
```

## 快速启动

### 前置要求

- Rust 工具链（stable，建议 1.75+）
- Node.js 18+
- npm 或 pnpm

### 1. 初始化 SQLite 数据库

```bash
cd backend
mkdir -p data
sqlite3 data/finance.db < init.sql
```

> 如果没有 sqlite3 命令，启动后端后首次请求会自动创建空库，但没有演示数据。

### 2. 启动后端服务（端口 8109）

```bash
cd backend
cargo run
```

服务启动后访问：`http://localhost:8109/api/health` 验证。

### 3. 启动前端服务（端口 3109）

```bash
cd frontend
npm install
npm run dev
```

前端启动后访问：`http://localhost:3109`

## 角色与权限边界

| 角色 | 可操作节点 | 权限 |
|------|-----------|------|
| 融资申请登记员 | 登记、补正 | 创建申请单、补正后重新提交、添加备注 |
| 融资申请审核主管 | 核验 | 核验通过、退回补正、添加备注 |
| 供应链金融平台复核负责人 | 复核归档 | 复核通过归档、退回、添加备注 |

**队列刷新前的权限校验：**
- 校验风控审核是否已处理
- 校验资金经理是否具备复核权限
- 异常则停在原队列，原因同步到页面和接口

## 业务模块

### 1. 融资申请单登记
- 登记员创建申请单，提交后进入待核验队列
- 自动分配当前处理人、节点截止时间

### 2. 过程核验
- 风控审核主管核验发票、合同等材料
- 核验通过 → 流转到复核队列
- 核验退回 → 打回登记员补正，记录异常原因

### 3. 复核归档
- 资金经理复核通过后归档，放款确认
- 可退回补正，流程回退

## 状态流转

```
草稿 → 待核验(登记完成) → 核验通过(核验完成) → 已归档(复核完成)
                ↓              ↑
            待补正(核验退回) ──┘
                ↓
            待补正(复核退回) → ...
```

## 到期预警

- **正常**：截止时间 > 1 天
- **临期**：截止时间 ≤ 1 天
- **逾期**：已超过截止时间

节点超时会显示责任人和预警等级。

## 批量处理规则

批量操作时系统会**逐条校验**，异常的申请单会被拦截，不会整批放行：

- 越权操作 → 拦截，返回"权限不足"
- 状态冲突 → 拦截，返回"状态冲突"
- 版本冲突 → 拦截，返回"版本冲突"
- 处理成功 → 状态流转 + 版本号 +1 + 记录日志

详情页会留下对应的补正动作和异常原因。

## 常见问题

**Q: 前端请求后端报 CORS 错误？**
A: 检查 `Rocket.toml` 端口是否为 8109，以及 `main.rs` 中 CORS 白名单是否包含 `http://localhost:3109`。

**Q: 数据库文件在哪？**
A: `backend/data/finance.db`，可直接用 sqlite3 客户端打开查看。

**Q: 如何重置演示数据？**
A: 删除 `backend/data/finance.db`，重新执行 `sqlite3 data/finance.db < init.sql`。
