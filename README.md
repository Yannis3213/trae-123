# 派出所-月底集中处理警情处置单系统

一个完整的警情处置单管理系统，支持角色权限控制、状态流转、批量处理、到期预警等功能。

## 技术栈

- **前端**: React 18 + TypeScript + Vite + Ant Design + Zustand
- **后端**: Rust + Rocket + SQLx
- **数据库**: SQLite
- **端口配置**: 前端 3107，后端 8107

## 端口说明

⚠️ **重要：本系统所有端口已固定配置，请勿修改**

- 前端端口: `3107` (Vite 开发服务器)
- 后端端口: `8107` (Rocket HTTP 服务器)
- CORS 白名单: `http://localhost:3107`, `http://127.0.0.1:3107`
- 前端代理: `/api` -> `http://localhost:8107`

## 快速开始

### 环境要求

- Rust >= 1.75.0
- Node.js >= 18.0.0
- SQLite 3 (系统自带，无需额外安装)

### 1. 克隆项目

```bash
git clone <repository-url>
cd trae-123-7
```

### 2. SQLite 数据库初始化

**数据库文件会在后端启动时自动创建和初始化，无需手动操作。**

数据库文件位置: `backend/data/police_case.db`

包含以下数据表:
- `users` - 用户表（包含3个演示账号）
- `cases` - 警情处置单表（10条演示数据）
- `attachments` - 附件表
- `processing_records` - 处理记录表
- `audit_notes` - 审计备注表

如需重置数据库:
```bash
rm -f backend/data/police_case.db
# 重启后端服务即可自动重建
```

### 3. 启动后端

```bash
cd backend
cp .env.example .env
cargo run --bin server
```

后端将在 `http://localhost:8107` 启动。

首次启动时会自动:
- 创建数据库文件
- 初始化所有数据表
- 插入3个演示用户和10条演示警情数据

### 4. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端将在 `http://localhost:3107` 启动。

### 5. 访问系统

打开浏览器访问: `http://localhost:3107`

## 系统功能

### 角色权限控制

| 操作 | 登记员 | 民警 | 所长 |
|------|--------|------|------|
| 新建警情登记 | ✅ | ❌ | ❌ |
| 补正材料提交审核 | ✅ | ❌ | ❌ |
| 处置派警 | ❌ | ✅ | ❌ |
| 回访确认 | ❌ | ✅ | ❌ |
| 退回补正 | ❌ | ❌ | ✅ |
| 办结归档 | ❌ | ❌ | ✅ |
| 添加审计备注 | ❌ | ✅ | ✅ |
| 查看列表/详情 | ✅ | ✅ | ✅ |

> **安全机制**: 后端 API 会校验当前用户角色，前端按钮隐藏不代表后端放行。直接调用接口时越权操作会返回 403 错误。

### 后端校验规则

每次状态变更时，后端会校验:

1. **角色权限**: 当前用户角色是否允许执行该操作
2. **处理人权限**: 是否为当前案件的指定处理人
3. **状态流转**: 当前状态是否允许变更到目标状态
4. **版本号**: 防止重复提交、覆盖他人修改
5. **证据完整性**:
   - 提交审核: 登记材料必须齐全
   - 移交复核: 登记材料、派及时限、回访证据必须齐全
   - 办结归档: 三项证据必须全部齐全
6. **逾期处理**: 逾期案件需先添加逾期说明

### 列表功能

- 多条件筛选: 状态、阶段、到期状态、关键词搜索
- 到期预警: 正常/临期/逾期三色分组显示
- 统计面板: 各状态、各阶段案件数量统计
- 批量处理: 选择多条案件批量变更状态，逐条返回成功/失败原因
- 证据完整性可视化: 登记材料、派及时限、回访证据三项状态一目了然

### 详情功能

- 案件基本信息完整展示
- 处理流程时间线: 接警员 → 民警 → 所队负责人，按时间顺序展示
- 证据完整性校验面板
- 附件列表: 分类展示登记材料、证据材料、回访材料
- 审计备注: 异常原因高亮显示
- 操作按钮: 根据角色和当前状态动态显示，未满足条件时显示禁用原因
- 版本号显示: 防止并发修改冲突

### 批量处理

- 支持按角色批量操作:
  - 登记员: 批量提交审核
  - 民警: 批量移交复核
  - 所长: 批量退回补正、批量办结
- 逐条校验，互不影响
- 结果展示: 成功/失败原因逐条列明
- 逾期案件拦截: 批量办结时逾期案件会被拦截，需到详情页单独处理

## API 接口文档

所有接口前缀: `/api`

### 认证接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/auth/login` | 登录获取 Token | 公开 |
| GET | `/auth/me` | 获取当前用户信息 | 已登录 |

### 案件接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/cases` | 获取案件列表 | 已登录 |
| GET | `/cases/:id` | 获取案件详情 | 已登录 |
| POST | `/cases` | 新建警情登记 | 登记员 |
| PUT | `/cases/status` | 更新案件状态 | 按角色 |
| POST | `/cases/batch` | 批量处理 | 按角色 |
| POST | `/cases/attachments` | 添加附件 | 已登录 |
| POST | `/cases/notes` | 添加审计备注 | 民警/所长 |
| GET | `/cases/:id/records` | 获取处理记录 | 已登录 |
| GET | `/cases/expiring` | 获取到期预警统计 | 已登录 |
| GET | `/statistics` | 获取全局统计 | 已登录 |

## 常见问题

### Q: 后端启动报错 "database is locked"
A: SQLite 是文件型数据库，请确保没有其他进程在访问数据库文件。

### Q: 前端跨域问题
A: 已在 Vite 配置代理 `/api` 到后端，同时后端 CORS 已配置白名单 `http://localhost:3107`。确保使用 `localhost` 而非 `127.0.0.1` 访问前端。

### Q: 如何重置所有数据？
A: 删除 `backend/data/police_case.db` 文件，重启后端服务即可。

### Q: 版本冲突如何处理？
A: 说明该案件已被其他用户修改，请刷新页面获取最新版本后再操作。
