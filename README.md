# 连锁药房-月底集中处理近效期处理单系统

近效期药品处理单管理系统，支持门店店员、执业药师、区域经理三种角色协同处理近效期药品。

## 技术栈

- **前端**：Next.js 14 App Router + TypeScript + Tailwind CSS
- **后端**：Go + Chi 框架
- **数据库**：SQLite（项目内置，自动初始化）

## 端口配置

- 前端端口：**3106**
- 后端端口：**8106**
- CORS 白名单：http://localhost:3106

> 端口号已在前端配置、后端监听、CORS 配置、README 启动命令中统一使用，请勿修改为其他端口。

## 快速开始

### 1. 启动后端

```bash
cd backend
go mod tidy
go build -o server .
./server
```

后端启动后会自动：
- 初始化 SQLite 数据库（位于 `backend/data/pharmacy.db`）
- 自动填充 seed 演示数据
- 监听 8106 端口

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端启动后访问：http://localhost:3106

## SQLite 初始化

系统首次启动时自动完成初始化：

1. 自动创建 `backend/data/pharmacy.db` 数据库文件
2. 自动创建所有数据表
3. 自动填充演示数据（用户、处理单、附件、处理记录等）
4. 后续启动会跳过已存在的 seed 数据

如需重置数据，删除 `backend/data/` 目录后重启后端即可。

## 四类异常样例

系统预置了 8 条演示处理单，涵盖以下四类异常场景：

### 1. 正常流转样例
- **JXQ-2026-0002（布洛芬缓释胶囊）- 处理中，待区域经理复核
- **JXQ-2026-0006（复方丹参片）** - 已关闭，完整流程示例

### 2. 缺材料异常
- **JXQ-2026-0003（维生素C片）** - 待派发，缺少全部证据材料
- **JXQ-2026-0008（藿香正气水）** - 待派发，缺少全部证据
- 缺少近效期巡检、调拨申请、下架确认时，无法提交复核和通过复核

### 3. 超时/逾期异常
- **JXQ-2026-0004（感冒灵颗粒）** - 已逾期2天，待执业药师处理
- **JXQ-2026-0007（蒙脱石散）** - 临期预警，待区域经理复核
- 到期预警队列中用颜色区分：绿色=正常、黄色=临期（3天内）、红色=逾期

### 4. 退回补正/状态冲突
- **JXQ-2026-0005（诺氟沙星胶囊）** - 已退回，待门店店员补正
- 退回后状态版本号机制防止重复提交和状态冲突

## 功能特性

### 角色边界与权限

- **门店店员**：创建处理单、补正材料、上传近效期巡检记录
- **执业药师**：处理近效期药品、调拨申请、下架确认、提交复核
- **区域经理**：复核通过/退回补正

### 状态流转

```
待派发 → 处理中 → 已关闭
  ↓        ↑
  已退回 ↙
```

- 待派发 (pending_dispatch) → 执业药师处理
- 处理中 (processing) → 区域经理复核
- 已关闭 (closed) → 流程结束
- 已退回 (returned) → 店员补正后重走流程

### 核心功能

1. **近效期处理单队列
   - 列表筛选（状态筛选、关键词搜索、只看我的
   - 到期预警（正常/临期/逾期一目了然
   - 批量选择和批量处理（按每条返回成功/失败原因

2. **详情办理
   - 证据材料（近效期巡检、调拨申请、下架确认
   - 处理记录时间线
   - 审计备注
   - 异常原因展示

3. **后端校验
   - 角色权限校验
   - 当前处理人校验
   - 状态流转校验
   - 版本号乐观锁
   - 必填证据校验
   - 越权、重复提交、状态冲突、旧版本提交、缺证据全部拦截

### 数据库表

- `users` - 用户表
- `near_expiry_orders` - 近效期处理单
- `attachments` - 附件/证据表
- `processing_records` - 处理记录表
- `audit_notes` - 审计备注表
- `exception_reasons` - 异常原因表

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/health | 健康检查 |
| GET | /api/users/me | 当前用户信息
| GET | /api/users | 用户列表
| GET | /api/stats | 统计数据
| GET | /api/orders/ | 处理单列表
| POST | /api/orders/ | 创建处理单
| GET | /api/orders/:id | 处理单详情
| POST | /api/orders/:id/process | 处理操作
| POST | /api/orders/:id/audit-notes | 添加审计备注
| POST | /api/orders/:id/attachments | 上传附件
| POST | /api/batch-process | 批量处理
