# 水务营业厅 - 月底集中处理开户申请系统

基于 Lit + Vite + Go (Chi) + SQLite 的水务营业厅开户申请管理系统。

## 技术栈

- **前端**: Lit 3 + Vite 5
- **后端**: Go 1.21 + Chi 5 + SQLite
- **认证**: JWT
- **端口**: 前端 3002，后端 8002

## 快速启动

### 前置条件

- Go 1.21+
- Node.js 18+
- npm 或 yarn

### 1. 启动后端服务

```bash
cd backend
go mod tidy
go run main.go
```

后端服务将在 `http://localhost:8002` 启动。

### 2. 启动前端服务

```bash
cd frontend
npm install
npm run dev
```

前端服务将在 `http://localhost:3002` 启动。

### 3. 访问系统

打开浏览器访问: `http://localhost:3002`

## SQLite 数据库初始化

数据库文件会在首次启动后端服务时自动创建在 `backend/data/water_biz.db`。

同时会自动初始化演示数据（用户、开户申请、处理记录等）。

如需重置数据库，删除 `backend/data/water_biz.db` 文件后重启后端服务即可。

## 功能模块

### 用户开户
- 窗口人员创建开户申请
- 录入申请人信息、安装地址、用水性质等
- 自动生成申请编号和到期日期

### 资料审核
- 抄表主管派发申请给处理人
- 审核申请材料（身份证、房产证等）
- 支持审核通过或退回补正

### 装表派工
- 录入水表编号和安装地址
- 记录装表异常情况
- 跟踪装表进度

### 营业经理复核
- 对完成装表的申请进行复核
- 复核通过则关闭申请
- 支持退回补正

### 到期预警
- **正常队列**: 到期时间 > 3天
- **临期队列**: 到期时间 ≤ 3天 且 > 今天
- **逾期队列**: 已超过到期日期
- 节点超时自动计算责任人

### 批量处理
- 批量派发（抄表主管）
- 批量关闭（营业经理）
- 逾期批量推进（抄表主管）
- 逐条展示成功/失败原因

### 审计轨迹
- 完整记录每个节点的操作
- 记录操作人、前后状态、操作时间
- 记录异常原因和备注

### 数据导出
- 支持 CSV 格式导出
- 只导出当前筛选结果
- 导出文件标注数据导出时间

## 状态流转

```
待派发 → 处理中 → 已关闭
   ↓        ↓
  (窗口)  (资料审核/装表派工)
          ↗   ↖
     退回补正  审核通过
```

- **待派发**: 窗口人员建单后初始状态
- **处理中**: 抄表主管派发后，包含资料审核、装表派工等子流程
- **已关闭**: 营业经理复核通过后终态

## 后端校验规则

后端接口会做以下校验，确保数据一致性：

1. **角色权限校验**: 每个操作验证当前用户角色
2. **处理人校验**: 只有当前处理人可以执行操作
3. **状态校验**: 验证当前状态是否允许执行该操作
4. **版本校验**: 使用乐观锁防止并发修改冲突
5. **必填证据校验**: 退回补正必须填写异常原因，装表必须填水表号
6. **越权拦截**: 未登录或无权限的操作直接拒绝
7. **重复提交防护**: 同一版本号只能提交一次

## API 接口

### 认证
- `POST /api/auth/login` - 登录
- `GET /api/auth/me` - 获取当前用户

### 开户申请
- `GET /api/applications/` - 列表（支持分页、筛选）
- `GET /api/applications/{id}` - 详情
- `POST /api/applications/` - 创建
- `PATCH /api/applications/{id}/status` - 更新状态
- `GET /api/applications/stats/warning` - 预警统计
- `GET /api/applications/export/csv` - 导出CSV
- `POST /api/applications/batch/process` - 批量处理
- `GET /api/applications/{id}/audit-logs` - 审计日志
- `POST /api/applications/{id}/remarks` - 添加备注

## 注意事项

1. 本系统为演示系统，密码为明文存储，生产环境请使用加密存储
2. JWT Secret 为硬编码，生产环境请通过环境变量配置
3. SQLite 适合单节点部署，高并发场景请考虑使用其他数据库
