# 直播电商团队 - 月底集中处理直播选品单系统

一个完整的直播选品单管理系统，支持角色接力办理、状态流转、批量处理、审计追溯和到期预警。

## 技术栈

- **前端**：Fresh (Deno + Preact + Twind)
- **后端**：Go + Echo
- **数据库**：SQLite（项目内本地文件）
- **前端端口**：3004
- **后端端口**：8004

## 快速启动

### 1. 启动后端服务

```bash
cd backend
go mod tidy
go run main.go
```

服务将在 **8004** 端口启动，数据库文件会自动创建在 `backend/data/live_selection.db`。

### 2. 启动前端服务

```bash
cd frontend
deno task start
```

服务将在 **3004** 端口启动，访问 http://localhost:3004 即可使用系统。

> 注意：前端需要安装 Deno 运行时，详见 [Deno 官网](https://deno.land/)。

## 状态流转

```
                    提交审核            审核通过            复核归档
草稿(draft) ────────────────→ 待审核(pending_audit) ──────────────→ 审核通过(audit_passed) ──────────────→ 已同步(synced)
     ↑                              │
     │                              │ 退回补正
     └──────────────────────────────┘
     退回补正(returned)
```

**状态说明：**
- **草稿**：登记员创建后初始状态，可编辑
- **待审核**：提交后进入审核，等待审核主管处理
- **审核通过**：审核通过后，等待复核负责人归档
- **已同步**：复核归档完成，流程结束
- **退回补正**：审核不通过，退回登记员补正材料

## 核心功能

### 列表筛选
- 按状态筛选：全部 / 草稿 / 待审核 / 审核通过 / 已同步 / 退回补正
- 关键词搜索：支持选品单号、商品名称模糊搜索
- 分页展示

### 详情办理
- 完整展示选品单基本信息
- 三个业务模块标签页，分别展示证据材料
- 显示上一处理人意见（黄色高亮）
- 显示异常原因和逾期原因
- 附件列表展示
- 审计轨迹时间线

### 批量处理
- 批量审核（审核主管）
- 批量归档（复核负责人）
- 批量结果逐条说明成功/失败原因
- 支持勾选多条单据批量操作

### 审计轨迹
- 记录所有状态变更
- 记录所有异常拦截
- 记录补正操作
- 记录附件上传
- 可追溯到"待审核""审核通过""已同步"等关键节点

### 到期预警队列
- 按责任人分组展示
- 一眼分清正常（绿色）、临期（黄色，≤3天）、逾期（红色）
- 支持逾期批量推进
- 节点超时按责任人计算

### 后端安全校验
所有操作在进入下一步前都会进行严格核验：

1. **角色权限校验** - 验证当前用户角色是否有权限执行操作
2. **责任人校验** - 验证当前处理人是否匹配
3. **状态校验** - 验证当前状态是否允许该操作
4. **版本校验** - 乐观锁，防止旧版本重复提交
5. **必填证据校验** - 每个模块有必填证据，缺项拦截
6. **处理意见校验** - 审核/复核必须填写处理意见
7. **截止时间校验** - 逾期单据特殊处理
8. **审计备注写入** - 所有状态变更、异常都记录审计备注

### 异常拦截
所有异常都会被拦截并写入审计备注，便于追溯：
- 越权操作
- 重复提交
- 状态冲突
- 旧版本提交
- 缺证据请求
- 超时逾期

## 数据库表结构

### live_selection_orders - 直播选品单主表
存储选品单基本信息、状态、证据、异常等。

### selection_attachments - 附件表
存储选品单相关的附件信息，按模块分类。

### process_records - 处理记录表
记录每次状态变更的操作人、意见、版本等。

### audit_remarks - 审计备注表
记录所有审计轨迹，包括状态变更、异常、补正等。

### users - 用户表
存储系统用户信息。

## 常见问题

### Q: 数据库文件在哪里？
A: 数据库文件位于 `backend/data/live_selection.db`，首次启动时自动创建。

### Q: 如何重置演示数据？
A: 删除 `backend/data/live_selection.db` 文件，重启后端服务即可重新生成演示数据。

### Q: 为什么切换角色后需要刷新页面？
A: 切换角色会重新登录并获取新的 token，刷新页面确保数据与新角色权限一致。

### Q: 批量处理失败怎么办？
A: 批量处理结果会逐条显示失败原因，可以根据失败原因修正后重新操作。常见失败原因包括状态不符、权限不足、缺证据等。

### Q: 临期和逾期怎么区分？
A: 距离截止时间 ≤ 3 天为临期（黄色），超过截止时间为逾期（红色）。
