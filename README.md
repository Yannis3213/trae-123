# 维修服务平台 - 月底集中处理维修订单系统

## 技术栈

- **前端**: Solid.js + Vite + TypeScript（端口 3109）
- **后端**: Go + Fiber 框架（端口 8109）
- **数据库**: SQLite（本地文件 `backend/repair.db`）

## 快速启动

### 1. 启动后端

```bash
cd backend
go run .
```

后端启动后监听 `http://localhost:8109`，首次运行自动初始化 SQLite 数据库并写入演示数据。

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端开发服务器启动在 `http://localhost:3109`，API 请求通过 Vite 代理转发到后端 8109 端口。

## SQLite 初始化

数据库文件为 `backend/repair.db`，首次启动后端时自动完成以下操作：

1. 创建 6 张数据表：`users`、`repair_orders`、`attachments`、`process_records`、`audit_notes`、`exception_reasons`
2. 插入 3 个演示用户和 8 条演示工单（含关联的流转记录、附件、审计备注、异常原因）

如需重置数据，删除 `repair.db` 文件后重新启动后端即可。

## 业务流程

### 角色接力

```
客服专员（建单）→ 师傅调度（接单→施工→完工）→ 服务经理（验收/退回→归档）
```

### 状态流转

```
待接单 → 已接单 → 施工中 → 待验收 → 验收通过 → 已归档
                              ↓
                         退回补正 → 已接单（重新接单）
```

### 三大业务模块

1. **维修订单登记**：客服专员创建工单，填写标题、描述、截止日期、优先级
2. **过程核验**：师傅调度接单、施工、完工（需上传施工证据）；服务经理验收（需验收证据）或退回补正
3. **复核归档**：服务经理验收通过后归档，退回时记录异常原因和补正要求

### 后端校验规则

| 校验项 | 说明 |
|--------|------|
| 角色校验 | 每个状态变更只允许对应角色操作 |
| 状态校验 | 当前状态必须匹配目标变更的前置状态 |
| 版本校验 | 乐观锁，提交版本必须与当前版本一致，冲突时拒绝并提示刷新 |
| 必填证据 | 完工需施工证据，验收通过需验收证据 |
| 越权拦截 | 非对应角色调用接口直接返回 403/409 |
| 重复提交 | 版本号机制防止重复提交 |

## 到期预警

| 分组 | 规则 | 列表颜色 |
|------|------|----------|
| 正常 | 截止日期 > 3天，或已完成/已归档 | 绿色 |
| 临期 | 截止日期在 3 天以内 | 橙色 |
| 逾期 | 截止日期已过且未完成 | 红色 |

列表页支持按到期状态筛选，逾期工单详情中可查看补正动作和异常原因。

## 端口配置

| 配置项 | 端口 | 位置 |
|--------|------|------|
| 前端开发服务器 | 3109 | frontend/vite.config.ts → server.port |
| 后端 API 服务器 | 8109 | backend/main.go → app.Listen |
| API 代理 | /api → :8109 | frontend/vite.config.ts → proxy |
| CORS 白名单 | http://localhost:3109 | backend/main.go → cors.Config |

## 项目结构

```
backend/
  main.go          # Fiber 入口，端口 8109，CORS 配置
  database.go      # SQLite 初始化、建表、种子数据
  models.go        # 数据模型、请求/响应结构体
  handlers.go      # 12 个 API 处理函数
  middleware.go     # JWT 认证中间件
  routes.go        # 路由注册

frontend/
  vite.config.ts   # Vite 配置，端口 3109，API 代理
  src/
    api.ts         # API 请求封装
    store.ts       # Solid.js 响应式状态管理
    types.ts       # TypeScript 类型定义
    App.tsx        # 主应用，hash 路由
    index.css      # 全局样式
    components/
      LoginPage.tsx    # 登录页 + 角色快捷登录
      Header.tsx       # 顶栏统计 + 用户信息
      OrderList.tsx    # 工单列表（状态Tab+到期筛选+批量选择）
      OrderDetail.tsx  # 工单详情（流转+附件+备注+操作）
      BatchProcess.tsx # 批量处理（逐条成功/失败结果）
      AuditTrail.tsx   # 审计轨迹时间线
```
