# 社区健身房 - 月底集中处理会员入会单系统

基于 Nuxt 3 + Litestar + SQLite 的社区健身房会员入会单集中处理系统。

## 端口约定

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 | 3109 | Nuxt 3 开发服务器 |
| 后端 | 8109 | Litestar API 服务 |

**所有配置均使用以上端口，请勿修改。**

## 技术栈

- **前端**: Nuxt 3 + TypeScript + Vue 3
- **后端**: Python 3.11+ + Litestar + SQLAlchemy
- **数据库**: SQLite（项目内置）
- **认证**: JWT Bearer Token

## 快速开始

### 1. 启动后端服务

```bash
cd backend

# 创建虚拟环境（首次）
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 初始化数据库（首次或重置数据）
python data/seed.py

# 启动服务
python -m uvicorn app.main:app --host 0.0.0.0 --port 8109 --reload
```

后端 API 文档（Swagger UI）: http://localhost:8109/schema/swagger

### 2. 启动前端服务

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问地址: http://localhost:3109

### 3. 一键启动（两个终端分别执行）

```bash
# 终端 1 - 后端
cd backend && source .venv/bin/activate && python -m uvicorn app.main:app --host 0.0.0.0 --port 8109 --reload

# 终端 2 - 前端
cd frontend && npm run dev
```

## 角色与权限

### 会员入会登记员
- 创建新的入会单
- 补正被退回的入会单
- 查看自己创建的入会单

### 会员入会审核主管
- 审核待核验的入会单
- 审核通过 → 进入复核阶段
- 审核退回 → 退回登记员补正
- 批量审核

### 社区健身房复核负责人
- 复核已审核的入会单
- 复核通过 → 归档完成
- 复核退回 → 退回登记员补正
- 查看会籍顾问和私教主管记录
- 批量复核

## 状态流转

```
待核验 ──审核通过──→ 待核验（待复核） ──复核通过──→ 核验完成
   │                      │
   └──审核退回──→ 核验失败 ←──复核退回──┘
                      │
                      └──补正提交──→ 待核验
```

三段状态：
- **待核验**: 等待审核或等待复核
- **核验失败**: 审核/复核退回，待补正
- **核验完成**: 复核通过，已归档

## 到期预警

系统按处理时限自动计算到期状态：

| 状态 | 说明 |
|------|------|
| 正常 | 剩余处理时间 > 24 小时 |
| 临期 | 剩余处理时间 ≤ 24 小时 |
| 逾期 | 已超过处理时限 |

到期状态在列表页和详情页均有展示，逾期单据会产生超期未处理异常。

## 四类异常样例

### 1. 资料缺失
- **样例**: 赵小美（核验失败）、郑小芳（待核验）
- **触发**: 入会单缺少三类证据中的任意一类
- **表现**: 列表页显示"有异常"标签，详情页显示异常记录
- **处理**: 登记员补正上传证据后重新提交

### 2. 状态冲突
- **样例**: 赵小美（审核退回）、孙大壮（逾期 + 状态冲突）
- **触发**: 审核/复核退回时，或对不符合状态的单据进行操作
- **表现**: 操作被拦截，返回状态冲突错误
- **处理**: 按流程在正确状态下操作

### 3. 越权推进
- **样例**: 需登录非对应角色操作触发
- **触发**: 
  - 登记员执行审核/复核操作
  - 审核主管执行复核操作
  - 补正非自己创建的单据
- **表现**: 操作被拦截，返回越权错误
- **处理**: 使用对应角色账号操作

### 4. 超期未处理
- **样例**: 王小强（待核验逾期）、孙大壮（核验失败逾期）
- **触发**: 入会单超过处理时限仍未处理
- **表现**: 列表页逾期标签，异常记录生成
- **处理**: 尽快处理，系统自动记录超时责任人

## API 接口

### 认证
- `POST /api/auth/login` - 登录
- `GET /api/auth/me` - 获取当前用户

### 入会单
- `GET /api/enrollments` - 获取入会单列表
- `GET /api/enrollments/{id}` - 获取入会单详情
- `POST /api/enrollments` - 创建入会单
- `POST /api/enrollments/audit` - 审核
- `POST /api/enrollments/review` - 复核
- `POST /api/enrollments/correct` - 补正
- `POST /api/enrollments/batch/audit` - 批量审核
- `POST /api/enrollments/batch/review` - 批量复核
- `GET /api/enrollments/stats/summary` - 统计概览
- `POST /api/enrollments/check-exceptions` - 检查队列异常

### 请求参数（列表）
| 参数 | 类型 | 说明 |
|------|------|------|
| status | string | 筛选状态: pending/failed/completed |
| expiry_status | string | 筛选到期状态: normal/approaching/overdue |
| keyword | string | 搜索关键词（姓名/电话/合同号） |
| my_todo | boolean | 只看我的待办 |
| page | int | 页码，默认 1 |
| page_size | int | 每页数量，默认 20 |

### 后端提交校验

所有状态变更接口都会校验：
- **角色权限**: 当前用户角色是否有操作权限
- **当前处理人**: 操作是否属于当前用户职责范围
- **状态校验**: 当前状态是否允许该操作
- **版本号**: 防止重复提交和旧版本覆盖
- **必填证据**: 审核通过需三类证据齐全

### 错误拦截

| 错误类型 | HTTP 状态 | 场景 |
|----------|-----------|------|
| 越权推进 | 403 | 角色无操作权限 |
| 重复提交 | 409 | 版本号不匹配 |
| 状态冲突 | 409 | 当前状态不允许此操作 |
| 资料缺失 | 400 | 审核通过时证据不全 |
| 超期未处理 | - | 系统自动标记，不直接拦截 |

## 前端功能

### 列表页（第一屏）
- 入会单队列为主视图
- 顶部统计卡片（总数、各状态、各到期状态）
- 状态筛选 Tab（待核验/核验失败/核验完成）
- 到期筛选（正常/临期/逾期）
- 关键词搜索、我的待办筛选
- 右侧边栏：证据摘要 + 快捷操作
- 批量选择和批量处理
- 刷新队列时自动检测异常

### 详情页
- 基本信息展示
- 证据材料卡片（三类证据，显示是否齐全）
- 处理进度时间线
- 操作记录时间轴（含操作人、状态变更、备注）
- 异常记录展示
- 侧栏操作区（按角色显示对应操作）
- 版本号显示

### 批量处理
- 列表页多选入会单
- 批量审核（审核主管）
- 批量复核（复核负责人）
- 每条结果分别返回成功/失败原因
- 批量退回需填写原因

### 角色切换
- 页面右上角一键切换演示账号
- 切换后自动刷新列表和权限

## 测试场景建议

1. **正常流转**: clerk1 创建 → supervisor1 审核通过 → lead1 复核通过
2. **缺材料**: 创建时不上传全部证据，审核时会被拦截
3. **超时/逾期**: 查看王小强、孙大壮等逾期单据
4. **退回补正**: supervisor1 审核退回 → clerk1 补正提交 → 再次审核
5. **状态冲突**: 尝试对已完成的单据进行操作
6. **越权操作**: 用 clerk1 账号尝试执行审核操作
7. **批量处理**: 选择多个单据进行批量审核/复核
8. **版本冲突**: 两个页面同时操作同一条单据
