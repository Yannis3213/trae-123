# 图书馆月底集中处理借阅记录系统

基于 **Next.js App Router (前端 3004) + Rust Actix Web (后端 8004) + SQLite 的借阅记录流转系统。

## 端口约定（全局共用，请勿修改）

| 服务 | 端口 | 说明 |
| --- | --- | --- |
| 前端 (Next.js) | **3004** | `npm run dev` 或 `next dev -p 3004` |
| 后端 (Actix Web) | **8004** | `127.0.0.1:8004` |
| 数据库 | SQLite 文件 | `backend/library_borrow.db` (首次启动自动创建) |
| CORS 白名单 | `http://localhost:3004` | 已配置 |
| 前端 API 地址 | `http://localhost:8004/api` | 通过 `NEXT_PUBLIC_API_BASE_URL` |

## 快速启动

### 1. 启动后端（Rust）

```bash
cd backend
cargo run
```

启动成功后监听：

- HTTP 地址：`http://127.0.0.1:8004`
- 首次启动会自动生成 SQLite：`backend/library_borrow.db`，并自动写入 5 位读者和 4 条示例借阅记录（覆盖「待分派 / 已转办 / 已回访等）。

### 2. 启动前端（Next.js）

```bash
cd frontend
npm install
npm run dev
```

启动成功后访问：`http://localhost:3004`。

## 角色与权限流转

| 角色 | 权限 | 可处理状态 | 可推进到 |
| --- | --- | --- | --- |
| 借阅登记员 | 发起 / 补正 | 退回补正 | 待分派 |
| 流通馆员 | 初始队列 | 待分派 | 已转办 / 退回补正 / 已逾期 |
| 采编馆员 | 中段处理 | 已转办 | 已回访 / 退回补正 |
| 借阅审核主管 | 办理 | 已回访 | 复核归档 / 退回补正 |
| 馆长 | 最终意见 / 归档 | 全部 | 全部 |

核心流转路径：

```
借阅登记员创建
       ↓ (待分派)
   流通馆员
       ↓ (已转办)
   采编馆员
       ↓ (已回访)
   借阅审核主管
       ↓ (复核归档)
     馆长
```

## 主要功能

- **角色切换：右上角下拉切换当前值班角色（用于测试）
- **借阅记录工作台：按当前角色过滤、状态、到期预警、关键字筛选
- **批量办理：逐条校验，成功/失败原因逐条展示，失败原因写入审计备注
- **详情办理：基础信息 / 办理历史 / 审计备注 / 责任追溯链
- **读者档案：查看读者并快捷发起新借阅
- **借阅登记：借阅登记员角色可创建记录
- **逾期处理：正常 / 临期 / 逾期 / 节点超时四类聚合处理
- **接口校验：在接口层校验当前角色、处理人、状态、版本、必填证据；越权、重复提交、状态冲突、旧版本提交、缺证据均不可通过
- **到期预警：正常 / 临期（≤3天）/ 逾期；节点超时显示责任人

## 接口校验清单

- 资料缺失：未提供所需证据（按状态转换所需材料清单时拦截
- 状态冲突：当前状态不可按该角色无权限推进时拦截
- 越权推进：角色与状态不匹配时拦截
- 版本冲突：提交版本与当前最新版本不一致时拦截
- 节点超时：非馆长角色推进超期未处理的记录时拦截
- 所有拦截均写入审计备注，可通过详情页「审计备注」中追溯

## 数据模型（SQLite）

- `readers：读者档案
- `borrow_records`：借阅记录（版本号、缺材料列表、当前处理人
- `attachments`：附件（预留结构
- `process_records`：办理历史（含必需证据要求与实际提供的证据）
- `audit_notes`：审计备注与异常原因（异常类型、异常详情）
