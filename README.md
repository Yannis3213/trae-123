# 行政后勤中心 - 月底集中处理会议预约单系统

## 系统概述

本系统实现会议预约单的月底集中处理，采用三角色三阶段流转机制：

### 角色配置
- **会议预约登记员（行政专员）**：发起/补正 - `zhangsan` 张三
- **会议预约审核主管（后勤主管）**：办理 - `lisi` 李四
- **行政后勤中心复核负责人（行政经理）**：复核归档 - `wangwu` 王五

### 流转状态
- **待签收**（pending_sign）：新创建或补正后待审核
- **异常回传**（exception_return）：审核不通过，退回补正
- **签收完成**（sign_complete）：全部环节审核通过，归档完成

### 三阶段证据核对
1. **会议室预约**：入口数据，需提供预约确认证据
2. **设备准备**：过程核对，需提供设备准备证据
3. **使用确认**：最终证据，需提供使用确认证据

| 层级 | 技术 | 端口 |

- 前端端口：`3001` - 见 `frontend/vite.config.js` 和 `frontend/package.json`

## 接口校验规则

### 权限校验
- 接口层校验 `X-Current-User` 请求头
- 校验当前角色是否匹配单据 `current_role`
- 校验当前用户是否为单据指定 `handler`
- 越权操作返回明确错误码和错误信息

### 状态校验
- 状态流转校验：待签收 → 可通过/退回；异常回传 → 可通过/退回；已完成 → 不可操作
- 版本校验：提交版本必须与数据库版本一致，防止并发冲突
- 重复提交：同一版本号只能提交一次

### 必填证据校验
- 会议室预约环节必须提供 `room_booking_evidence`
- 设备准备环节必须提供 `equipment_evidence`
- 使用确认环节必须提供 `usage_evidence`

### 批量处理校验
- 批量处理前逐条校验：责任人、截止时间、处理意见、审计备注
- 逾期单据必须填写审计备注
- 退回/异常回传必须填写异常原因
- 批量结果逐条返回成功/失败原因，不整批放行

## 测试场景

### 正常流转测试
1. 登录 **张三（行政专员）** 创建会议预约单
2. 切换到 **李四（后勤主管）** 签收，补充会议室预约证据，审核通过
3. 进入设备准备环节，补充设备准备证据，审核通过
4. 进入使用确认环节，补充使用确认证据，审核通过
5. 切换到 **王五（行政经理）** 复核归档

### 异常场景测试

**1. 缺材料拦截**
- 登录李四，尝试不填证据直接审核通过 → 接口返回 `MISSING_EVIDENCE`

**2. 版本冲突**
- 打开两个详情页，在A页修改并提交
- 在B页使用旧版本号提交 → 接口返回 `VERSION_CONFLICT`

**3. 越权操作**
- 登录张三，尝试处理应由李四处理的单据 → 接口返回 `WRONG_ROLE`

**4. 状态冲突**
- 尝试对已完成的单据执行退回操作 → 接口返回 `STATUS_CONFLICT`

**5. 逾期处理**
- 找到逾期单据（红色标识），尝试审核通过不填审计备注 → 接口返回 `OVERDUE_REQUIRES_REMARK`

**6. 批量拦截**
- 选择多条单据（包含已完成单据）执行批量通过 → 已完成单据被逐条拦截并说明原因

**7. 退回补正**
- 李四在待签收状态异常回传给张三
- 张三补正材料后重新提交
- 李四再次审核通过

## 数据库结构

SQLite 数据库文件：`backend/meeting_booking.db`

### 核心表

| 表名 | 说明 |
|------|------|
| `meeting_orders` | 会议预约单主表 |
| `attachments` | 附件表 |
| `process_records` | 处理记录表（审计追踪） |
| `audit_remarks` | 审计备注表 |
| `exception_reasons` | 异常原因表 |

## 操作记录可反推责任

所有操作均记录 `process_records` 表，包含：
- 操作人、操作角色、操作时间
- 操作前状态/环节、操作后状态/环节
- 处理意见、审计备注、异常原因
- 单据版本号
- 是否为异常操作

详情页的"处理记录"区域以时间线形式完整展示所有操作轨迹。

## 到期预警规则

- **正常**：截止时间 > 24小时
- **临期**：0 < 截止时间 ≤ 24小时（黄色标识）
- **逾期**：截止时间 ≤ 0（红色标识）

列表页和详情页均显示预警状态，逾期单据处理需要审计经理特别备注。
