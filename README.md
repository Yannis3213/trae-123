# 航空地服-月底集中处理值机记录系统

## 技术栈

- **前端**: Angular 17 + Vite(esbuild)，端口 3001
- **后端**: Go + Echo，端口 8001
- **数据库**: SQLite（本地文件 `backend/data/aviation.db`，自动创建）

## 快速启动

### 1. 后端

```bash
cd backend
go mod tidy
go run main.go
```

后端启动后监听 `http://localhost:8001`，首次启动会自动：
- 创建 `data/aviation.db`（SQLite 数据库）
- 执行表结构迁移（6 张表：users、checkin_records、attachments、processing_records、audit_notes、exception_reasons）
- 插入演示数据

### 2. 前端

```bash
cd frontend
npm install
npm start
```

前端启动后访问 `http://localhost:3001`，API 请求通过代理转发到后端 8001 端口。

## SQLite 初始化

数据库文件在 `backend/data/aviation.db`，首次运行后端时自动创建和初始化。如需重置，删除 `data/` 目录后重新启动后端即可。

6 张数据表：
| 表名 | 说明 |
|------|------|
| users | 用户账号 |
| checkin_records | 值机记录主表 |
| attachments | 附件（旅客值机/行李托运/异常交接证据） |
| processing_records | 处理记录（操作日志） |
| audit_notes | 审计备注 |
| exception_reasons | 异常原因 |

## 角色接力流程

```
值机员 ──提交审核──→ 行李主管 ──审核通过──→ 站点经理 ──确认同步──→ 完成
   ↑                    │
   └──── 退回补正 ←─────┘
```

- **值机员**: 创建记录 → 提交审核（草稿→待审核）；补正提交（退回→待审核）
- **行李主管**: 审核通过（待审核→审核通过，需值机+行李证据）；退回补正（待审核→退回，需填写原因）
- **站点经理**: 确认同步（审核通过→已同步，需全部3类证据）

## 后端安全校验

即使绕开页面直接调用 API，后端也会拦截以下非法请求：
- **越权操作**: 当前角色不匹配当前处理环节
- **状态冲突**: 记录状态不允许当前操作
- **版本冲突**: 乐观锁校验，旧版本提交被拒绝
- **缺证据请求**: 审核通过需值机+行李证据；确认同步需全部3类证据
- **重复提交**: 版本号自增，防止重复操作

## 预警队列

按截止时间分类：
- **正常**（绿色）: 距截止时间 > 72 小时
- **临期**（橙色）: 距截止时间 24~72 小时
- **逾期**（红色）: 距截止时间 < 24 小时或已过期
