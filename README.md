# 园区招商中心-月底集中处理招商线索单系统

## 项目概述

本系统是为园区招商中心设计的月底集中处理招商线索单系统，支持招商线索单的全流程管理，包括线索登记、审核、复核、归档等环节，同时支持批量处理和审计追踪。

## 技术栈

### 前端
- Vue 3 + Vite
- Vue Router 4
- Pinia 状态管理
- Element Plus UI 组件库
- Axios HTTP 客户端
- Day.js 日期处理

### 后端
- Node.js
- Hono Web 框架
- SQLite 数据库（better-sqlite3）
- JWT 身份认证
- Bcrypt 密码加密
- CORS 跨域支持

## 快速开始

### 1. 安装依赖

```bash
# 安装根目录依赖（用于 concurrently）
npm install

# 安装前后端依赖
npm run install:all
```

### 2. 初始化数据库

```bash
cd backend
npm run init:db
```

### 3. 启动服务

#### 方式一：同时启动前后端（推荐）

```bash
# 在项目根目录执行
npm run dev
```

#### 方式二：分别启动

```bash
# 启动后端（端口 8109）
npm run dev:backend

# 启动前端（端口 3109）
npm run dev:frontend
```

### 4. 访问系统

打开浏览器访问：http://localhost:3109

## 角色权限说明

### 1. 招商线索登记员 (registrar)
- 查看自己创建的线索单
- 创建新线索单（草稿→待提交）
- 提交审核（待提交→待审核）
- 重新提交（已退回→重新提交）
- 查看退回原因和处理记录

### 2. 招商线索审核主管 (auditor)
- 查看待审核、重新提交的线索单
- 查看自己作为当前处理人的线索单
- 审核通过（待审核/重新提交→待复核）
- 退回补正（待审核/重新提交→已退回）

### 3. 园区招商中心复核负责人 (reviewer)
- 查看待复核、已通过、已拒绝、已归档的线索单
- 查看所有招商专员和招商主管的记录
- 复核通过（待复核→通过）
- 复核拒绝（待复核→拒绝）
- 归档（通过/拒绝→已归档）
- 添加审计备注

## 四类异常样例

系统内置了四类异常样例，用于测试不同场景的处理：

### 1. 缺材料 (missing_material)
- **线索单**：ZS20240601003（新能源汽车配件公司拜访跟进）
- **异常说明**：缺少企业营业执照附件和场地使用证明
- **测试场景**：审核主管退回补正，登记员重新提交时需补充材料

### 2. 逾期未处理 (overdue)
- **线索单**：
  - ZS20240601003（已退回，逾期1天）
  - ZS20240601006（待提交，逾期3天）
  - ZS20240601009（待复核，逾期5天）
  - ZS20240601011（待提交，临期1天）
- **异常说明**：超过截止时间未处理
- **测试场景**：
  - 列表页到期预警分组（正常/临期/逾期）
  - 批量处理时逾期线索单自动拦截
  - 详情页显示逾期提醒，需单独处理

### 3. 状态冲突 (status_conflict)
- **异常日志**：ZS20240601004 中记录了状态冲突的尝试
- **异常说明**：尝试从"已退回"直接跳转到"待复核"，违反状态流转规则
- **测试场景**：后端自动校验状态流转，非法操作被拦截并记录异常日志

### 4. 版本冲突 (version_conflict)
- **异常日志**：ZS20240601002 中记录了版本冲突的尝试
- **异常说明**：提交的版本号与当前版本号不匹配，可能存在并发修改
- **测试场景**：后端乐观锁机制，旧版本提交被拦截，需刷新后重试

## 主要功能模块

### 1. 线索单列表
- 多维度筛选（状态、优先级、类型、到期状态、关键词）
- 到期预警分组（正常/临期/逾期）
- 显示责任人、优先级、截止时间、当前处理人、异常标签
- 批量选择和批量推进
- 统计卡片展示各状态数量

### 2. 线索单详情
- 基本信息展示
- 附件列表（企业资料、拜访记录、签约合同）
- 处理结果和退回原因
- 处理记录时间轴
- 审计备注（仅审核/复核人员可见）
- 异常日志（仅作为证据）
- 办理操作区域（根据角色和状态显示可用操作）
- 状态流转说明

### 3. 批量处理
- 选择多个线索单批量推进
- 队列刷新前核对当前角色、当前状态、必填附件
- 逾期线索单自动拦截
- 逐条返回成功/失败原因
- 批量结果页面展示详细处理结果
- 支持跳转到详情页补正

### 4. 审计追踪
- 完整的处理记录（操作人、时间、状态变更、备注）
- 审计备注（审核/复核人员添加）
- 异常日志（状态冲突、版本冲突、缺材料、逾期）
- 所有操作留痕，可追溯

## 状态流转

```
草稿 → 待提交 → 待审核 → 待复核 → 通过 → 归档
               ↓        ↓        ↓
             重新提交   已退回   拒绝
               ↑        ↓
               └────────┘
```

### 状态说明
| 状态 | 说明 | 下一可能状态 |
|------|------|--------------|
| 草稿 | 登记员创建中 | 待提交 |
| 待提交 | 已完成，待提交审核 | 待审核 |
| 待审核 | 审核主管处理中 | 待复核、已退回 |
| 已退回 | 材料不全，需补正 | 重新提交 |
| 重新提交 | 补正后重新提交 | 待复核、已退回 |
| 待复核 | 复核负责人处理中 | 通过、拒绝 |
| 通过 | 复核通过，待签约 | 归档 |
| 拒绝 | 复核拒绝 | 归档 |
| 归档 | 流程结束 | - |

## 后端校验机制

### 1. 权限校验
- 每个接口校验用户角色权限
- 越权操作返回 403

### 2. 状态机校验
- 严格按照状态流转规则校验
- 非法状态转换被拦截

### 3. 处理人校验
- 校验当前处理人是否为当前用户
- 非处理人无法操作

### 4. 乐观锁（版本校验）
- 每次提交携带版本号
- 版本不匹配拦截，防止并发修改冲突

### 5. 必填附件校验
- 不同状态要求不同的必填附件
- 缺材料操作被拦截

### 6. 逾期拦截
- 批量处理时自动检查是否逾期
- 逾期线索单必须进入详情页单独处理

## API 接口列表

### 认证接口
- `POST /api/auth/login` - 登录
- `POST /api/auth/logout` - 登出
- `GET /api/auth/me` - 获取当前用户信息

### 线索单接口
- `GET /api/clues` - 获取线索单列表（支持筛选）
- `GET /api/clues/stats` - 获取统计数据
- `GET /api/clues/:id` - 获取线索单详情
- `POST /api/clues/:id/process` - 处理线索单
- `POST /api/clues/batch` - 批量处理
- `GET /api/clues/batch/:batchNo` - 获取批量处理结果
- `POST /api/clues/:id/audit-notes` - 添加审计备注
- `GET /api/clues/:id/abnormal-logs` - 获取异常日志

## 注意事项

1. **端口一致性**：前端端口 3109，后端端口 8109，所有配置文件中统一使用，请勿修改其他端口
2. **CORS 白名单**：仅允许 http://localhost:3109 访问后端 API
3. **SQLite 本地运行**：数据库文件位于 `backend/data/zhaoshang.db`，无需额外安装数据库服务
4. **异常日志仅作为证据**：不能替代详情页的真实处理结果
5. **批量处理拦截逾期**：逾期线索单必须进入详情页单独处理
6. **版本号递增**：每次成功操作后版本号自动 +1，防止并发修改冲突
7. **角色边界**：队列刷新前核对当前角色、当前状态和必填附件
