## 启动命令

```bash
# 后端
cd backend
python -m venv .venv && source .venv/bin/activate   # 首次
pip install -r requirements.txt                        # 首次
BACKEND_PORT=8106 FRONTEND_PORT=3106 python main.py

# 前端（新开终端）
cd frontend
pnpm install                                           # 首次
pnpm dev
```

浏览器打开 <http://localhost:3106>，任意账号登录。

## 本地 SQLite 数据说明

- **路径**：`backend/training_projects.db`，若存在则跳过初始化，删除文件即重置演示数据。
- **建表**：后端启动时 `models.init_db()` 自动 `CREATE TABLE IF NOT EXISTS`。
- **演示数据**：首次建表后 `init_data.seed_demo_data()` 写入 3 个账号 + 8 条项目单 + 附件、处理记录、审计备注、异常记录。
- **标签可追溯**：`audit_notes.note_content` 与 `exception_records.exception_message` 均内嵌中文标签 + code（如 `[补正/supplement]`、`[缺少必备证据]`、`[节点超时/逾期]`、`[状态冲突]`），直接打开 DB 即可追溯异常原因与责任岗位。

## 后端强制拦截规则（统一进审计备注 + 异常记录）

- **PERM-001 越权**：非当前处理角色 / 非创建人尝试操作。
- **VER-001 版本冲突**：提交 `version` 与 DB 不一致（乐观锁）。
- **FLOW-* 状态冲突**：在不允许的状态执行动作（如草稿态直接复核）。
- **EVI-* 缺少必备证据**：需求/方案/合同三阶段必备附件或字段缺失；补正提交缺少清单；退回未填原因（`EVI-RJ-001`）。
- **DUE-* 节点逾期**：截止日期已过仍未推进。
- **DUP-001 重复提交**：同一动作在当前状态重复触发。
