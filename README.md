# 美容连锁门店 - 月底集中处理美容预约单系统

基于 React + Vite（前端，端口 3107）和 Rust + Poem（后端，端口 8107）构建，SQLite 本地持久化。

## 端口约定

| 服务   | 端口  |
| ------ | ----- |
| 前端   | 3107  |
| 后端   | 8107  |

前后端所有配置（Vite proxy、后端监听、CORS 白名单、API 基础地址）均使用上述端口。

## 启动方式

### 1. 后端（Rust + Poem）

```bash
cd backend
cargo run
```

服务启动后监听 `http://localhost:8107`，Swagger UI 地址：`http://localhost:8107/`

### 2. 前端（React + Vite）

```bash
cd frontend
npm install
npm run dev
```

浏览器访问：`http://localhost:3107`

## 功能概览

### 角色
- **护理师 (beautician)**：新建预约单、提交复核、补正后重新提交
- **美容顾问 (consultant)**：复核通过 / 退回补正
- **门店店长 (store_manager)**：归档、退回补正、批量操作

### 预约单状态
- `draft` 草稿
- `pending_review` 待复核
- `archived` 已归档

### 样例数据（首次启动自动写入 SQLite）

| 单号 | 顾客 | 状态 | 异常 | 到期状态 |
|------|------|------|------|----------|
| MR20260601001 | 张美丽 | 草稿 | - | 正常 |
| MR20260601002 | 刘优雅 | 待复核 | 缺材料 | 临期 |
| MR20260601003 | 王小姐 | 待复核 | 逾期 | 已逾期 |
| MR20260601004 | 赵太太 | 待复核 | 退回补正 | 正常 |
| MR20260601005 | 孙女士 | 已归档 | - | 正常 |

### 核心特性

1. **列表看板**：三栏分布（正常 / 临期 24h / 逾期），到期预警一目了然
2. **角色切换**：顶部下拉切换身份，按钮按角色动态显示，护理师不能替店长归档
3. **详情办理**：
   - 基本信息 + 状态 + 到期状态
   - 顾客预约 / 项目确认 / 服务回访 三类证据摘要
   - 异常原因、补正说明独立展示
   - 补正处理记录和完整审计轨迹（时间线）
4. **批量处理**：批量提交复核、复核通过、归档、退回补正，结果逐条展示成功/失败原因
5. **后端校验**：
   - 角色权限校验
   - 当前处理人校验
   - 状态机校验（不能跳环节、不能悄悄推进）
   - 版本号乐观锁（重复提交 / 旧版本提交拦截）
   - 必填证据校验（异常只允许补正或退回）

## 数据持久化（SQLite）

首次启动自动创建 `backend/beauty_appointments.db`，包含：

- `beauty_appointments` 美容预约单主表
- `appointment_attachments` 证据附件
- `audit_trails` 审计轨迹
- `processing_records` 处理记录（补正动作等）
