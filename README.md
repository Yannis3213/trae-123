# 外贸公司-月底集中处理外贸订单系统

## 系统概述

外贸订单月底集中处理系统，基于角色的工作流处理外贸订单，涵盖客户询盘、报价确认、订单签订到归档的完整流程。

## 技术栈

- **前端**: Nuxt 3 + Vue 3 (端口: 3108)
- **后端**: Python + Django Ninja + SQLite (端口: 8108)
- **数据库**: SQLite（项目内本地文件）

## 岗位与角色

| 角色编码 | 角色名称 | 主要职责 |
|---------|---------|---------|
| clerk | 外贸登记员 | 发起订单、录入询盘、补正资料 |
| supervisor | 外贸审核主管 | 办理报价确认阶段 |
| reviewer | 外贸公司复核负责人 | 复核订单签订、归档 |

## 订单状态

- `待派发` (pending_dispatch): 新创建的订单，等待派发
- `处理中` (processing): 正在某个岗位处理中
- `已关闭` (closed): 已归档完成

## 流程阶段

1. `客户询盘` (inquiry) → 外贸登记员处理
2. `报价确认` (quote_confirmation) → 外贸审核主管处理
3. `订单签订` (order_signing) → 外贸公司复核负责人处理
4. `已归档` (archived)

> **重要规则**: 报价确认、订单签订信息不齐时，外贸订单停在原队列，不能推进到下一阶段。

## 启动方式

### 后端 (端口 8108)

```bash
cd backend

# 创建虚拟环境并安装依赖（首次）
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

# 初始化数据库（首次）
python manage.py makemigrations orders
python manage.py migrate
python manage.py init_seed_data   # 初始化示例数据

# 启动服务
python manage.py runserver 0.0.0.0:8108
```

后端 API 文档: http://localhost:8108/api/docs

### 前端 (端口 3108)

```bash
cd frontend

# 安装依赖（首次）
npm install

# 启动开发服务
npm run dev
```

前端访问: http://localhost:3108

## 功能特性

### 列表页
- 顶部角色切换（外贸登记员/外贸审核主管/外贸公司复核负责人）
- 统计卡片：总数、待派发、处理中、已关闭、异常、逾期
- 筛选条件：状态、阶段、优先级、预警（正常/临期/逾期）、异常、关键词搜索
- 列表列：订单编号、客户、产品、金额、状态、阶段、优先级、责任人、当前处理人、截止时间、预警、异常标签
- 支持多选批量处理
- 新建订单（仅外贸登记员）

### 详情页
- **业务内容 Tab**: 客户询盘、报价确认、订单签订（含确认开关）
- **附件 Tab**: 上传/下载附件，按阶段分类
- **办理结果 Tab**: 记录办理结果说明
- **退回原因 Tab**: 查看退回补正原因
- **异常原因 Tab**: 查看/添加异常原因和补正动作
- **审计备注 Tab**: 查看/添加审计备注
- **处理轨迹 Tab**: 完整时间线，记录每个操作的人、角色、状态变化、版本、证据情况
- **办理操作 Tab**: 根据当前角色显示可用操作（提交、派发、办理、复核、退回、补正、归档）

### 批量处理
- 支持派发、办理、复核、关闭归档操作
- 逐条处理，每条记录成功/失败原因
- 逾期或缺资料订单逐条拦截，不整批放行

### 接口安全机制
- **角色校验**: 每个请求校验 X-User-Role 头
- **版本校验**: 乐观锁，旧版本提交返回 409
- **状态机校验**: 非法状态流转被拦截
- **证据校验**: 办理/复核/归档必须确认已上传证据
- **完整性校验**: 报价确认、订单签订信息不齐时拦截

## 预警规则

| 预警级别 | 判断条件 |
|---------|---------|
| 正常 | 无截止时间 或 截止时间 > 当前+2天 |
| 临期 | 当前时间 < 截止时间 ≤ 当前+2天 |
| 逾期 | 截止时间 < 当前时间 |
