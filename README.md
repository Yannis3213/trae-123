# B2B批发平台-月底集中处理商家入驻单系统

## 系统概述

本系统是一个完整的B2B批发平台商家入驻单处理闭环系统，支持多角色协作、状态流转、证据校验、批量处理和到期预警。

### 技术栈

- **前端**: Next.js 14 App Router + Ant Design 5.x
- **后端**: Node.js + Fastify 4.x
- **数据库**: SQLite (better-sqlite3)
- **前端端口**: 3107
- **后端端口**: 8107

### 岗位角色

| 角色 | 用户名 | 密码 | 职责 |
|------|--------|------|------|
| 商家入驻登记员 | registrar | 123456 | 发起入驻、补正材料 |
| 商家入驻审核主管 | auditor | 123456 | 资质审核、办理入驻单 |
| B2B批发平台复核负责人 | leader | 123456 | 复核归档、逾期批量推进 |

## 快速启动

### 1. 安装依赖

```bash
# 后端依赖
cd backend
npm install

# 前端依赖
cd ../frontend
npm install
```

### 2. 初始化数据库

```bash
cd backend
npm run init-db
```

### 3. 导入演示数据

```bash
npm run seed
```

### 4. 启动服务

```bash
# 启动后端服务 (端口: 8107)
cd backend
npm start

# 启动前端服务 (端口: 3107)
cd ../frontend
npm run dev
```

### 5. 访问系统

打开浏览器访问: http://localhost:3107

## 核心功能

### 1. 角色视角分离

- **平台招商（登记员）**: 看入驻登记队列，发起入驻、补正材料
- **商家运营（审核主管）**: 看资质审核队列，审核资质、办理入驻单
- **平台负责人**: 看全部队列，复核归档、逾期批量推进

### 2. 业务模块

- **商家入驻**: 入驻申请、材料提交、签收确认
- **资质审核**: 资质核验、异常回传、审核通过/驳回
- **商家入驻单登记**: 信息登记、材料补正、提交复核

### 3. 状态流转

```
待签收 → 签收完成 → 审核中 → 审核通过/异常回传 → 入驻登记 → 复核中 → 已归档
     ↓          ↓          ↓
  已取消    退回补正    审核驳回
```

### 4. 到期预警

- **正常**: 处理期限 > 1天
- **临期**: 处理期限 ≤ 1天（黄色预警）
- **逾期**: 已超过处理期限（红色预警）

### 5. 批量处理

- 批量审核、批量登记
- 逐条返回成功/失败原因
- 逾期批量推进（仅平台负责人）

### 6. 安全防护

- 角色权限校验
- 处理人身份校验
- 状态机流转校验
- 乐观锁版本控制（防止并发冲突）
- 证据材料完整性校验

### 7. 异常分类

| 异常类型 | 说明 |
|----------|------|
| material_missing | 材料缺失 |
| permission_denied | 权限不足 |
| timeout | 超时/逾期 |
| status_conflict | 状态冲突 |
| version_conflict | 版本冲突 |
| duplicate_submit | 重复提交 |
| evidence_missing | 证据缺失 |

## 数据库表结构

1. **users** - 用户表
2. **merchant_entry_forms** - 商家入驻单主表
3. **attachments** - 附件表
4. **processing_records** - 处理记录表
5. **audit_notes** - 审计备注表
6. **exception_reasons** - 异常原因表
7. **batch_results** - 批量处理结果表

## 业务流程测试场景

### 场景1：正常流转

1. 登录 registrar → 签收 ME20250101 → 提交审核
2. 登录 auditor → 审核 ME20250102 → 审核通过
3. 登录 registrar → 登记 ME20250103 → 提交复核
4. 登录 leader → 复核归档

### 场景2：缺材料停住

1. 登录 auditor → 尝试审核 ME20250104
2. 系统提示：缺税务登记证、银行开户许可证
3. 退回补正 → 登录 registrar → 补正材料 → 重新提交

### 场景3：超时/逾期停住

1. 查看 ME20250105（已逾期）
2. 普通角色无法处理逾期单据
3. 登录 leader → 逾期批量推进 → 重新分配处理人

### 场景4：状态冲突停住

1. 两个用户同时打开同一份单据
2. 先提交的用户成功，版本号+1
3. 后提交的用户提示：版本冲突，请刷新页面

### 场景5：越权操作拦截

1. 登录 registrar → 尝试直接复核归档
2. 系统提示：权限不足，只有平台负责人可复核
3. 绕开页面直调接口也会被后端权限中间件拦截

## API接口

### 认证接口

- `POST /auth/login` - 登录
- `GET /auth/me` - 当前用户信息
- `GET /auth/users` - 用户列表

### 入驻单接口

- `GET /forms` - 入驻单列表
- `GET /forms/:id` - 入驻单详情
- `POST /forms` - 创建入驻单
- `POST /forms/:id/operation` - 操作入驻单
- `GET /forms/constants` - 常量字典
- `GET /forms/statistics` - 统计数据
- `POST /forms/:id/audit-notes` - 添加审计备注

### 批量处理接口

- `POST /batch/process` - 批量处理
- `POST /batch/promote-overdue` - 逾期批量推进
- `GET /batch/results/:batchNo` - 批量结果查询

### 预警接口

- `GET /alerts/deadline` - 到期预警
- `GET /alerts/exceptions` - 异常记录
- `GET /alerts/statistics` - 预警统计
