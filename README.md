# 小贷公司 · 月底集中处理借款申请单系统

以贷后主管复核视角设计的借款申请单集中处理系统。涵盖信贷员录入、风控核验、贷后主管审批的全流程，支持批量处理、到期预警、审计轨迹追踪。

## 技术栈

- **前端**: Angular 17 + Vite（端口 3004）
- **后端**: Python Flask（端口 8004）
- **数据库**: SQLite（本地文件）

> 前后端端口、CORS 白名单、请求地址统一使用 **前端 3004 / 后端 8004**，请勿随意修改。

## 快速开始

### 1. 后端启动

```bash
cd backend

# 安装依赖
pip install -r requirements.txt

# 初始化数据库 + 演示数据
python seed_data.py

# 启动服务（端口 8004）
python app.py
```

后端服务启动后访问：http://localhost:8004/api/health

### 2. 前端启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器（端口 3004）
npm start
```

前端访问地址：http://localhost:3004

## 核心功能

### 第一屏：借款申请单队列

- 左侧主列表：按创建时间倒序展示所有借款申请单
- 右侧证据摘要：借款申请、资料核验、审批放款三节点的必填证据概览
- 顶部到期预警卡片：正常 / 临期 / 逾期 三队列独立统计，点击可筛选
- 筛选：按状态、节点、到期状态、关键词搜索
- 批量勾选 + 批量处理

### 详情页

- 基本信息卡片
- 办理操作区（根据当前角色和状态显示不同操作按钮）
- 证据摘要侧栏（三节点齐全度实时显示）
- 附件列表
- 异常原因列表
- 补正动作说明
- 处理记录时间线（审计轨迹）
- 审计备注（可追加）

### 批量处理

- 风控可批量核验（通过 / 失败 / 退回补正）
- 贷后主管可批量审批（通过 / 拒绝 / 退回补正）
- **逐条校验**：每笔单据独立校验，异常单据拦截并给出原因
- 批量结果页：成功/失败统计 + 逐条明细 + 失败原因

### 到期预警

- **正常队列**: 到期日 3 天以上
- **临期队列**: 到期日 3 天内（含 3 天）
- **逾期队列**: 已超过到期日
- 列表行背景色区分：临期橙、逾期红
- 顶部统计卡点击可快捷筛选

## 后端校验机制

即使绕开前端直接调接口，后端也会进行以下校验。所有单笔与批量操作均通过统一函数 `validate_application_action` 执行一致的业务规则：

### 1. 角色权限校验
- 信贷员：只能创建、提交、补正重提
- 风控：只能执行核验操作（通过/失败/退回）
- 贷后主管：只能执行审批操作、完成放款

### 2. 状态机校验
每个动作只允许在特定状态下执行：
- `DRAFT / CORRECTION_REQUIRED` → 提交 → `PENDING_VERIFICATION`
- `PENDING_VERIFICATION` → 核验通过 → `VERIFICATION_PASSED`
- `PENDING_VERIFICATION` → 核验失败 → `VERIFICATION_FAILED`
- `PENDING_VERIFICATION` → 退回补正 → `CORRECTION_REQUIRED`
- `VERIFICATION_PASSED` → 审批通过 → `APPROVED`
- `VERIFICATION_PASSED` → 审批拒绝 → `REJECTED`
- `VERIFICATION_PASSED` → 退回补正 → `CORRECTION_REQUIRED`
- `APPROVED` → 放款完成 → `COMPLETED`

### 3. 版本乐观锁
- 每条申请单带 `version` 字段
- 提交操作时需带上当前版本号
- 版本不匹配返回 409 状态冲突，防止重复提交、旧版本覆盖

### 4. 当前处理人 / 创建人校验
- 只有 `current_handler` 匹配当前登录用户的角色才能执行对应动作
- 信贷员补正重提时，需校验是否为单据创建人
- 批量处理时每笔单据独立校验，越权单据被逐条拦截

### 5. 必填证据校验（三节点累积）
每个节点有对应的必填证据，操作前校验；推进到下一个节点时会累积校验前置节点的证据：

| 节点 | 必填证据 |
|------|----------|
| 借款申请 | 身份证、收入证明 |
| 资料核验 | 征信报告、核验记录 |
| 审批放款 | 审批意见、放款凭证 |

### 6. 超时校验
- 核验超时的单据，不允许「通过」操作
- 只能退回补正或标记失败

### 7. 异常留痕
所有拦截均写入 `exception_reasons` 表，类型包括：
- `MISSING_EVIDENCE` - 缺证据
- `VERSION_CONFLICT` - 版本冲突
- `TIMEOUT` - 超时
- `VERIFICATION_FAILED` - 核验失败
- `RETURNED` - 退回补正
- `REJECTED` - 审批拒绝

## 数据库表结构

### users（用户表）
用户账号、角色、姓名。

### loan_applications（借款申请单）
申请单主表，包含：
- 基本信息（申请人、金额、期限、用途等）
- 状态、当前节点、当前处理人
- 创建人、版本号、核验到期日、到期日

### attachments（附件 / 证据）
每笔申请单的附件/证据，关联节点和是否必填。

### processing_records（处理记录 / 审计轨迹）
每次状态变更都会写一条记录，包含：
- 动作类型、源状态、目标状态
- 处理人、处理人角色、处理节点
- 备注、时间戳

### audit_notes（审计备注）
贷后主管等可在任意节点追加审计备注。

### exception_reasons（异常原因）
系统自动记录或人工录入的异常信息，包含：
- 异常类型、原因、详情
- 发现人、发现时间
- 解决时间、解决方案

## 月底复核流程建议

1. 以 **贷后主管（supervisor_01）** 身份登录
2. 查看顶部到期预警：先处理逾期和临期单据
3. 点击「逾期」卡片，查看逾期列表
4. 进入详情页，审查：
   - 证据摘要是否齐全
   - 异常原因有哪些
   - 处理记录时间线是否合规
   - 节点超时责任人是谁
5. 对批量单据进行批量审批，查看逐条拦截结果
6. 在「处理记录」时间线确认每笔单据的完整流转轨迹
