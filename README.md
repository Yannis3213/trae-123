# 公关传播团队 - 月底集中处理传播计划单系统

## 端口说明

| 服务 | 端口 | 地址 |
|------|------|------|
| 前端 (SolidStart) | 3107 | http://localhost:3107 |
| 后端 (NestJS) | 8107 | http://localhost:8107 |
| CORS 白名单 | - | http://localhost:3107 |

## 快速启动

### 1. 启动后端

```bash
cd backend
npm install
npm run dev
```

后端启动后会自动创建 SQLite 数据库文件 `pr_system.db`。

### 2. 初始化演示数据

后端启动后，调用种子接口：

```bash
curl -X POST http://localhost:8107/api/plans/seed
```

或者在前端列表页点击「初始化演示数据」按钮。

### 3. 启动前端

```bash
cd frontend
npm install
PORT=3107 npm run dev
```

访问 http://localhost:3107

## SQLite 数据库

- 数据库文件：`backend/pr_system.db`
- 首次启动时由 TypeORM 自动创建表结构（`synchronize: true`）
- 删除 `pr_system.db` 后重启后端可重建数据库
- 重建后需重新调用 `/api/plans/seed` 初始化演示数据

## 核心功能

- **角色驱动办理**：三个角色各司其职，操作按钮按角色动态显示，各角色有独立的"我的待办"队列
- **状态流转闭环**：待签收 → 审核中 → 待复核 → 签收完成，异常分支：退回补正、异常回传
- **复核四重拦截**：资料缺失、状态冲突、越权推进、超期未处理四类拦截，状态冲突时保留原值
- **批量逐条校验**：批量签收/复核逐条校验，逾期不可整批放行，失败记录留审计轨迹和异常原因
- **到期预警分级**：正常(绿)、临期(橙)、逾期(红)，逾期显示责任人
- **审计轨迹完整**：每次操作留记录，含操作人、角色、时间、状态变更、退回原因、异常原因、审计备注
- **版本乐观锁**：防止旧版本重复提交，状态冲突时提示需要谁补正
- **数据持久化**：SQLite 持久化传播计划单、附件、处理记录、审计备注、异常原因，刷新后数据一致

## 我的待办队列

每个角色登录后都有独立的待办队列：

| 角色 | 待办内容 |
|------|----------|
| 传播计划登记员 | 退回补正、异常回传的单据（需补正后重新提交） |
| 传播计划审核主管 | 待签收、审核中的单据 |
| 公关传播团队复核负责人 | 待复核的单据 |

## 后端校验规则

后端接口校验，不依赖前端按钮，直接调接口也会被拦截：

1. **角色校验**：当前角色无权操作 → 403 Forbidden
2. **处理人校验**：当前用户不是当前处理人 → 拦截
3. **状态校验**：当前状态不允许此操作 → 409 Conflict，保留原值
4. **版本校验**：请求版本与最新版本不一致 → 409 Conflict
5. **证据校验**：复核归档时必填附件未上传 → 400 Bad Request，提示需登记员补正
