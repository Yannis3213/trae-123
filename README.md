# 电子元器件工厂 - 月底集中处理物料变更单系统

基于 **SvelteKit 前端** + **Python + Django Ninja 后端** + **SQLite** 的物料变更单闭环管理系统。

---

## 📌 端口约定

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 (SvelteKit) | **3005** | http://localhost:3005 |
| 后端 (Django Ninja) | **8005** | http://localhost:8005 |
| API 前缀 | - | http://localhost:8005/api/ |

> 前端请求地址、后端监听端口、CORS 白名单均使用以上端口，无其他硬编码端口。

---

## 🚀 快速启动

### 1. 后端启动

```bash
cd backend

# 安装依赖
pip install -r requirements.txt

# 数据库迁移（SQLite 自动创建）
python manage.py migrate

# 初始化演示数据
python manage.py init_demo_data

# 启动后端服务（端口 8005）
python manage.py runserver 0.0.0.0:8005
```

后端启动后访问：
- API 文档：http://localhost:8005/api/docs
- Admin 后台：http://localhost:8005/admin

### 2. 前端启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务（端口 3005）
npm run dev
```

前端启动后访问：http://localhost:3005

## 📋 角色与权限边界

### 状态流转图

```
待提交(draft) → BOM待确认 → BOM已确认 → 替代待核对 → 替代已核对
    ↑              ↓            ↓            ↓            ↓
    |           可退回       可退回        可退回        可退回
    |              ↓            ↓            ↓            ↓
已退回 ←--------←-----←-------←----←-------←----←------←--+
  ↓
重新提交 → (进入下一节点)
... → 待试产验证 → 试产已通过 → 待主管审核 → 主管审核通过
       ↓            ↓           ↓            ↓
     可退回       可退回       可退回        可退回
       ↓            ↓           ↓            ↓
... → 待生产经理复核 → 生产经理复核通过 → 待工厂复核 → 已归档
        ↓                  ↓               ↓
      可退回             可退回           可退回
```

### 角色可见范围

| 角色 | 可见范围 |
|------|----------|
| 登记员 | 自己创建的 + 待提交/已退回/重新提交状态的单据 |
| 物料员 | BOM 确认、替代核对、试产、审核阶段的单据 |
| 品质工程师 | 替代核对、试产、审核、生产经理复核阶段 |
| 审核主管 | **全部**单据 |
| 生产经理 | **全部**单据（可查看物料员 + 品质工程师记录） |
| 工厂复核负责人 | **全部**单据 |

### 操作权限校验

后端提交时强制校验：
- ✅ **当前角色**：角色是否有该操作权限
- ✅ **当前处理人**：是否为当前单据的处理人
- ✅ **状态**：当前状态是否允许该操作
- ✅ **版本号**：乐观锁，防止重复提交/旧版本提交
- ✅ **必填证据**：关键节点需对应证据材料
- ❌ 越权操作 → 拦截
- ❌ 重复提交 → 拦截
- ❌ 状态冲突 → 保留原值，提示需谁补正
- ❌ 旧版本提交 → 拦截，提示刷新

## 🔔 到期预警

系统自动按截止时间将单据分为三组：

| 分组 | 条件 | 标识色 |
|------|------|--------|
| 正常 | 截止时间 > 24 小时 | 🟢 绿色 |
| 临期 | 截止时间 ≤ 24 小时且未超时 | 🟡 黄色 |
| 逾期 | 已超过截止时间 | 🔴 红色 |

- 节点超时自动落到**当前处理人**名下
- 逾期自动生成**异常记录**，标注责任人
- 列表页可按预警状态筛选
- 点击「刷新预警」按钮可手动触发全量重算

## 📦 批量处理

- 支持列表多选后批量操作
- 不同角色看到不同的批量操作按钮
- **批量结果逐条返回**，包含成功/失败状态和原因
- 逾期批量推进时**逐条拦截**，失败单据可点击进入详情补正

## 🧪 四类异常样例

初始化演示数据后，系统包含以下典型场景样例：

### 1️⃣ 正常流转样例

| 单据号 | 状态 | 说明 |
|--------|------|------|
| MCO202506010001 | 待提交 | 草稿状态，可提交 |
| MCO202506010006 | 已归档 | 全流程完成的样例 |
| MCO202506010007 | 待主管审核 | 流转中，可继续推进 |

### 2️⃣ 缺材料样例

| 单据号 | 状态 | 异常点 |
|--------|------|--------|
| MCO202506010002 | BOM待确认 | 缺少 BOM 变更证据，无法确认 |

**验证方式**：
1. 用 `material`（物料员）登录
2. 尝试点击「确认BOM变更」
3. 系统提示「缺少确认BOM变更所需的证据材料」
4. 上传 BOM 证据后才能成功提交

### 3️⃣ 超时/逾期样例

| 单据号 | 预警 | 当前处理人 | 说明 |
|--------|------|------------|------|
| MCO202506010003 | 🔴 逾期 | 张登记（登记员） | 已退回，登记员未及时补正 |
| MCO202506010004 | 🔴 逾期 | 王品质（品质工程师） | 替代核对完成，但逾期未推进 |
| MCO202506010005 | 🟡 临期 | 王品质（品质工程师） | 待试产验证，即将到期 |

**验证方式**：
1. 用对应角色登录
2. 列表页查看预警分组统计
3. 进入详情查看「异常记录」标签页
4. 可看到超时异常记录和责任人

### 4️⃣ 退回补正 / 状态冲突样例

| 单据号 | 状态 | 说明 |
|--------|------|------|
| MCO202506010003 | 已退回 | 试产阶段被退回，需补正后重新提交 |
| MCO202506010008 | 重新提交 | 已重新提交，等待下一节点处理 |

**验证方式（退回补正）**：
1. 用 `quality`（品质工程师）登录
2. 打开待试产验证的单据
3. 点击「退回」按钮，填写退回原因
4. 用 `registrar`（登记员）登录
5. 看到单据变为「已退回」状态
6. 点击「补正修改」，更新信息后保存
7. 点击「重新提交」，状态变为「重新提交」

**验证状态冲突（版本冲突）**：
1. 同时打开两个详情页窗口
2. 在窗口 A 提交操作，版本号 +1
3. 在窗口 B 再提交操作
4. 系统提示「版本冲突：当前版本为 X，你提交的是版本 Y，请刷新后重试」

## 🗄️ SQLite 数据结构

主要数据表：

| 表名 | 说明 |
|------|------|
| `material_change_order` | 物料变更单主表 |
| `attachment` | 附件 |
| `processing_record` | 处理记录（操作日志） |
| `audit_remark` | 审计备注 |
| `exception_record` | 异常原因记录 |
| `bom_change_record` | BOM 变更记录 |
| `material_substitute_record` | 物料替代记录 |
| `pilot_verify_record` | 试产验证记录 |
| `user_profile` | 用户角色资料 |

刷新后列表、详情、统计、操作记录保持数据一致。
