# 城市公交公司-月底集中处理发车计划系统

## 项目概述

城市公交公司月底集中处理发车计划系统，实现发车计划从登记、审核到复核归档的全流程闭环管理。系统严格按角色分工推进，异常状态只允许补正或退回，杜绝越权与静默推进。

## 技术栈

- **前端**: Astro 5.x + React 18 Islands + TailwindCSS 3 + Zustand + Lucide Icons
- **后端**: Node.js + Koa 2.x + @koa/router + koa-body + @koa/cors
- **数据库**: SQLite3 (better-sqlite3)，项目内种子数据自动初始化
- **端口**: 前端 3002，后端 8002

## 快速启动

```bash
# 1. 安装后端依赖并启动
cd backend
npm install
npm run dev
# 后端运行在 http://localhost:8002

# 2. 安装前端依赖并启动（新终端）
cd frontend
npm install
npm run dev
# 前端运行在 http://localhost:3002

# 3. 打开浏览器访问
# http://localhost:3002
```

## SQLite 初始化

数据库文件位于 `backend/data/dispatch.db`，首次启动后端时自动创建表结构和种子数据。如需重新初始化，删除 `dispatch.db` 文件后重启后端即可。

```bash
# 重置数据库
rm backend/data/dispatch.db
# 重启后端，数据库将自动重建
cd backend && npm run dev
```

## 异常样例

### 1. 缺材料拦截
以审核主管身份操作 DP-2026-002，点击「办理」，后端返回：
```
缺少必要凭证：必须同时具备车辆排班表和司机签收单
```

### 2. 越权推进拦截
以登记员身份尝试审核待审核计划，后端返回：
```
无权限执行此操作
```

### 3. 版本冲突拦截
两人同时操作同一计划，后提交者收到：
```
版本冲突，请刷新后重试
```

### 4. 逾期批量拦截
选择逾期计划进行批量推进，结果中逐条说明：
```
DP-2026-003: 失败 - 当前状态 reviewing 无法推进（因缺必要凭证或越权）
```

### 5. 重复提交拦截
同一处理人对同一计划重复操作，后端返回：
```
重复操作：同一处理人不能对同一计划执行相同操作两次
```

### 6. 补正后重新流转
以登记员身份对 DP-2026-004 点击「补正」，填写补正说明后提交，计划重新进入待审核状态。

## API 接口

### 认证
- `POST /api/auth/login` - 登录
- `GET /api/auth/me` - 获取当前用户

### 发车计划
- `GET /api/plans` - 获取计划列表（支持 role/status/expiry 筛选）
- `GET /api/plans/:id` - 获取计划详情（含附件、处理记录、审计备注）
- `POST /api/plans` - 新建计划
- `PUT /api/plans/:id/advance` - 推进计划（含角色/状态/凭证/版本校验）
- `PUT /api/plans/:id/correct` - 补正退回件
- `PUT /api/plans/:id/reject` - 退回
- `POST /api/plans/batch-advance` - 批量推进（逐条返回成功/失败原因）

### 统计
- `GET /api/stats/expiry` - 到期预警统计（正常/临期/逾期）
- `GET /api/stats/queue` - 队列统计
- `GET /api/evidence/summary` - 证据摘要统计

## 业务流程

```
发车登记员新建计划(草稿) → 提交审核(待审核) → 发车审核主管办理(审核中) → 审核通过(待复核) → 复核负责人复核归档(已归档)
                                                    ↓                              ↓
                                                 退回(已退回) ← 登记员补正 → 重新提交
```

## 核心校验规则

1. **资料缺失检查**: 从待审核推进时必须同时具备车辆排班表和司机签收单
2. **状态冲突检查**: 当前状态不允许该操作则拒绝
3. **越权推进检查**: 当前角色无权执行该操作则拒绝
4. **超期未处理检查**: 截止日期已过标记逾期，到期预警面板分组显示
5. **版本冲突检查**: 旧版本提交被拒绝
6. **重复提交检查**: 同一处理人不可重复提交
