## 1. 产品概述

公关传播团队月底集中处理传播计划单系统，面向公关传播团队的传播计划登记员、传播计划审核主管和复核负责人，实现传播计划单从发起、审核、复核归档的全流程闭环管理。系统以角色驱动流程，以状态管控节点，以审计轨迹保障合规，重点解决月底集中处理时出现的材料缺失、状态冲突、超期未处理和越权推进等异常场景。

- 目标用户：公关传播团队（客户经理发起、内容审核处理、项目总监确认）
- 核心价值：月底集中处理时高效流转、异常拦截、合规审计、批量操作与逐条校验

## 3. 核心流程

### 3.1 传播计划单状态流转

传播计划单状态：草稿 → 待签收 → 审核中 → 待复核 → 已归档，异常分支：异常回传、退回补正。

1. 登记员发起传播计划单（状态：待签收）
2. 审核主管签收办理（状态：审核中），可退回补正（状态：退回补正→待签收）
3. 审核主管审核完成（状态：待复核）
4. 复核负责人复核归档（状态：已归档），异常时回传（状态：异常回传→待签收）

### 3.2 复核拦截规则

复核按钮点击后依次检查：
1. **资料缺失**：必填字段/附件未上传 → 拦截，提示需要谁补正
2. **状态冲突**：单据已被他人处理，状态已变更 → 保留原值，提示需要谁补正
3. **越权推进**：当前角色无权操作此状态 → 拦截
4. **超期未处理**：超过截止时间 → 标记逾期，留下异常记录

### 3.3 批量处理规则

- 批量签收/复核：逐条校验角色、处理人、状态、版本、必填证据
- 逾期批量推进：不能整批放行，逐条拦截，失败的在详情留下补正动作和异常原因
- 批量结果：逐条列成功/失败原因

### 3.5 到期预警规则

| 预警级别 | 判断条件 | 显示标识 |
|----------|----------|----------|
| 正常 | 截止时间 > 3天 | 绿色标识 |
| 临期 | 截止时间 ≤ 3天且 > 0天 | 橙色标识 |
| 逾期 | 截止时间 ≤ 当前时间 | 红色标识，显示责任人 |

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝灰 (#1a2332) + 金色点缀 (#c9a84c)，体现公关传播的专业感与权威感
- 辅助色：状态色（绿/橙/红）用于到期预警和异常标签
- 按钮风格：圆角4px，主操作按钮深蓝底白字，危险操作红色底
- 字体：思源黑体/Noto Sans SC，标题16-18px，正文13-14px，辅助信息12px
- 布局：左侧导航 + 顶部角色栏 + 右侧主内容区，卡片式列表
- 图标风格：线性图标（Lucide风格），简洁专业

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 登录页 | 登录表单 | 居中卡片，深色背景，三个演示账号按钮，角色图标 |
| 列表页 | 筛选区 | 顶部横向筛选条，下拉选择+日期范围 |
| 列表页 | Tab栏 | 传播计划/素材审核/投放确认，下划线切换 |
| 列表页 | 列表主体 | 卡片列表，每行：单据编号、标题、标签、责任人、截止时间、预警图标、操作按钮 |
| 列表页 | 批量操作栏 | 底部浮动栏，勾选计数+批量按钮 |
| 详情页 | 基本信息区 | 顶部卡片，网格布局展示字段 |
| 详情页 | 附件区 | 附件列表，缺失项红色标记 |
| 详情页 | 操作区 | 底部固定操作栏，按角色显示按钮 |
| 详情页 | 审计轨迹 | 左侧时间线，右侧操作详情 |
| 批量结果 | 结果列表 | 表格式，成功绿色✓，失败红色✗+原因 |

### 4.3 响应式

- 桌面优先设计，最小宽度1280px
- 列表在小屏下隐藏次要列，保留单据编号、标题、状态、操作
- 详情页信息区在窄屏下改为纵向堆叠
