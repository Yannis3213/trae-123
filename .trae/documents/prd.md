## 1. 产品概述

新能源汽车充电站月底集中处理设备巡检单系统，用于管理充电桩巡检、故障上报和设备巡检单的全生命周期流转。系统以设备巡检单为核心，串联充电桩巡检与故障上报两个上游模块，实现创建→处理→复核的完整审批链路，并支持退回补正、到期预警和审计追溯。

- 目标用户：站点值班员、运维工程师、运营经理三类角色
- 核心价值：确保月底设备巡检单集中处理的规范性、时效性和可追溯性

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 站点值班员 | 系统预设账号 | 创建设备巡检单、补正缺项材料 |
| 运维工程师 | 系统预设账号 | 处理设备巡检单（填写处理结果、上传证据） |
| 运营经理 | 系统预设账号 | 复核设备巡检单（通过或退回） |

### 2.2 功能模块

1. **设备巡检单列表**：状态筛选、到期预警色标、角色权限过滤、批量操作入口
2. **设备巡检单详情**：完整处理记录、附件预览、审计备注、补正操作、异常原因
3. **批量处理结果**：逐条成功/失败原因展示，回写状态
4. **审计轨迹**：状态变更时间线、操作人、审计备注追溯
5. **充电桩巡检**：创建巡检记录，关联设备巡检单
6. **故障上报**：创建故障记录，关联设备巡检单
7. **设备巡检单登记**：值班员创建新单，关联巡检和故障记录
8. **到期预警队列**：正常（绿）、临期（黄）、逾期（红）三色标识

## 3. 核心流程

设备巡检单从创建到归档的完整流转路径：

1. 站点值班员创建设备巡检单（状态：待提交），关联充电桩巡检和故障上报记录
2. 值班员提交设备巡检单（状态：待处理），进入运维工程师处理队列
3. 运维工程师处理设备巡检单，填写处理结果和证据（状态：待复核）
4. 运营经理复核，通过则完成（状态：已完成），退回则回到待处理（状态：已退回）
5. 被退回的单据，值班员可补正后重新提交（状态：重新提交→待处理）
6. 任意环节超时则标记逾期，逾期批量推进需逐条校验

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝灰（#1e293b）作为导航和标题底色，搭配翡翠绿（#10b981）作为通过/正常色，琥珀黄（#f59e0b）作为临期色，珊瑚红（#ef4444）作为逾期/错误色
- 按钮风格：圆角（8px），主操作按钮填充色，次要操作按钮描边
- 字体：思源黑体 Noto Sans SC，标题 18px/600，正文 14px/400，辅助文字 12px/400
- 布局风格：顶部导航 + 左侧菜单 + 右侧内容区，卡片式内容组织
- 图标风格：线性图标（Lucide），2px 描边

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 设备巡检单列表 | 筛选栏 | 下拉选择器、日期选择器、角色标签 |
| 设备巡检单列表 | 列表表格 | 带色标的行、状态徽章、操作按钮 |
| 设备巡检单详情 | 处理区域 | 表单卡片、文件上传区、提交按钮 |
| 设备巡检单详情 | 审计时间线 | 左侧时间轴、右侧内容卡片 |
| 批量处理结果 | 逐条结果 | 成功绿勾/失败红叉、原因文字 |
| 到期预警队列 | 三色列表 | 绿/黄/红色行背景、倒计时标签 |

### 4.3 响应式设计

桌面优先设计，内容区最小宽度 1024px，表格在小屏幕下支持横向滚动
