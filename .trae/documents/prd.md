## 1. 产品概述
产业园物业月底集中处理企业报修单系统，围绕企业报修诉求从登记、核验到归档全流程运转，解决报修单散落在通知和导出中无法集中管控的问题。
- 目标用户：产业园物业的企业客服、工程主管、园区经理
- 核心价值：报修单全生命周期闭环管理，状态可追溯、异常可拦截、到期可预警、责任可反推

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 企业客服 | 系统分配 | 登记报修单、提交报修单、退回补正后重新提交 |
| 工程主管 | 系统分配 | 处理报修单、过程核验、派单工程人员、退回报修单 |
| 园区经理 | 系统分配 | 复核报修单、归档报修单、退回报修单 |

### 2.2 功能模块
1. **报修单登记**：企业客服创建和提交报修单，填写诉求信息、上传证据附件
2. **过程核验**：工程主管处理报修单、派单工程人员、核验处理结果
3. **复核归档**：园区经理复核处理结果、归档关闭报修单
4. **报修单台账**：全量报修单列表，按诉求线索筛选，追踪工程派单和企业确认状态
5. **到期预警**：按正常/临期/逾期分组，节点超时落到责任人
6. **批量处理**：批量推进报修单，逐条返回成功/失败原因

## 3. 核心流程

### 3.1 报修单状态流转
企业客服创建报修单(待提交) → 企业客服提交(待处理) → 工程主管处理(处理中) → 工程主管核验(待复核) → 园区经理复核(待归档) → 园区经理归档(已归档)

任意环节可退回：当前处理人退回 → 报修单变为已退回 → 原提交人重新提交 → 回到退回前环节

### 3.2 状态定义
- 待提交：企业客服已创建但未提交
- 待处理：已提交等待工程主管处理
- 处理中：工程主管已受理正在处理
- 待核验：工程主管已处理等待核验
- 待复核：已核验等待园区经理复核
- 待归档：已复核等待归档
- 已归档：流程结束
- 已退回：被退回等待补正
- 重新提交：补正后重新提交进入流转

```mermaid
    "待归档" --> "归档" --> "已归档"

    "处理中" --> "退回" --> "已退回"
    "待核验" --> "退回" --> "已退回"
    "待复核" --> "退回" --> "已退回"
    "待归档" --> "退回" --> "已退回"
    "已退回" --> "重新提交" --> "待处理"
```

### 3.4 角色权限约束
- 企业客服：只能登记、提交、重新提交退回的报修单
- 工程主管：只能处理、核验待处理/处理中/待核验的报修单，不能替园区经理归档
- 园区经理：只能复核、归档待复核/待归档的报修单，不能跳过核验环节

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深靛蓝(#1e3a5f) + 琥珀黄(#f59e0b)警示色
- 辅助色：石板灰(#64748b)中性色、翡翠绿(#10b981)成功色、玫红(#ef4444)异常色
- 按钮风格：圆角8px，主要操作实色填充，次要操作描边
- 字体：Noto Sans SC，标题16px/14px，正文13px，辅助文字12px
- 布局：左侧固定导航 + 右侧内容区，顶部角色切换栏
- 图标：Lucide React图标库

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 报修单列表页 | 列表筛选区 | 多条件筛选栏、状态标签色块、到期预警分组卡片 |
| 报修单列表页 | 列表主体 | 数据表格、行内状态徽标、操作按钮、批量勾选框 |
| 报修单列表页 | 批量操作栏 | 批量推进/退回按钮、逐条结果弹窗 |
| 报修单详情页 | 状态横幅 | 当前状态大字展示、可执行操作按钮组 |
| 报修单详情页 | 处理时间线 | 垂直时间线、操作人头像、动作标签、附件预览 |
| 报修单详情页 | 异常面板 | 红色边框卡片、异常原因、补正建议 |
| 报修单登记页 | 表单区 | 分组表单、附件上传区、提交/保存按钮 |
| 报修单台账页 | 台账表格 | 宽表格、诉求摘要列、状态追踪列、派单状态列 |

### 4.3 响应式
- 桌面优先设计，最小宽度1280px
- 列表页在中等屏幕下筛选栏可折叠
- 详情页在窄屏下信息区纵向堆叠

### 4.4 到期预警视觉
- 正常：翡翠绿标签
- 临期(3天内到期)：琥珀黄标签+闪烁动画
- 逾期：玫红标签+加粗+红色背景条纹
